# 学员档案 Schema

> 学员档案是 Barry 职场英语工具箱的 single source of truth。
> 4 个原子 Skill 各自负责一个 section 的产出与更新；专属训练 Skill 以此档案为基础生成。

## 默认存储位置

`~/.barry-english/profile.md`

> 用 home 目录下隐藏文件夹存放，跨 AI Agent project 持久；学员若想多档（家庭成员/团队）可手动复制为不同文件名，由 onboarding 时指定路径。

## Schema (v0.4 · 2026-05-07 第二次校准)

### Section 1 · 个人基本情况

由 `barry-profile` Skill 产出/更新。

| 字段 | 必填 | 说明 |
|---|---|---|
| 昵称 | ✓ | 学员希望被怎么称呼 |
| 工作年限 / 行业 / 职位 | ✓ | 简短描述 |
| 英文使用场景 | ✓ | 至少 2-3 个具体场景，要带例子，不要"开会、写邮件"这种笼统答案 |
| 自述痛点 | ✓ | 用学员原话，带情绪、带场景 |
| 学习目标 | ✓ | 短期 (3 个月) + 长期 (1 年)，具象到画面 |
| 每日可投入时间 | ✓ | 含时间段 + 现实可行性评估 |
| 之前英语学习经历 | ✓ | 含教训（哪些方法有效/无效/为何放弃） |

### Section 2 · 目前水平（听口分项 + 综合）

由 `barry-assessment` Skill 产出/更新。

| 字段 | 必填 | 说明 |
|---|---|---|
| **口语 CEFR** | ✓ | A2 / B1 / B2 / C1 / C2 — 来自 Step 2 中翻英晋级制（典型 2-4 句）。注：测的是笔头主动转化能力，口头流利度需训练阶段持续校准 |
| **听力 CEFR** | ✓ | A1 / A2 / B1 / B2 / C1 / C2 — 来自 Step 3 五题阶梯式 |
| **听口综合 CEFR** | ✓ | 取口语 / 听力较低值（中国学员普遍口语 < 听力，综合应贴近口语） |
| 官方 CEFR 描述 | ✓ | 直接引用综合级别的 CEFR 官方描述 1-2 段 |
| 能力描述 | ✓ | 3-5 条"能做 / 不能做"，基于学员实际表现 |
| 评估方式 | ✓ | 标化考试参考 + 中翻英晋级制（典型 2-4 句） + 听力 5 题阶梯（留痕方便后续校准） |
| 评估时间 | ✓ | YYYY-MM-DD（标化考试要打折扣，重新评估时间也要追溯） |

> **关键**：分项给"口语 / 听力 / 综合"三个数字，避免合成单一 CEFR 时丢失"中国学员普遍口语 < 听力"这个核心训练抓手。下游 Solution 设计需读取分项数字，定向补弱。

### Section 3 · 卡点

由 `barry-assessment` Skill 同步产出/更新（与 Section 2 一起）。

#### 听力卡点（按"两步链路"诊断，不按 MCES 节点）

| 字段 | 必填 | 说明 |
|---|---|---|
| 阶段 | ✓ | a (听不清，S→E 卡) / b (听不懂，E→C/M 卡) / 混合 |
| 阶段 a 子表现 | 当阶段=a 或 混合时必填 | [单词熟但听不出 / 整体模糊 / N/A] |
| 具体例子 | ✓ | 学员自述的具体卡住场景 |

> **方法论依据**：听不懂分两步——阶段 a "你只能听到你知道的东西"（声音-文字匹配 + 储备）；阶段 b "看着不懂就听不懂"（语法支撑 + 英到中转换速度）。

#### 发音 S 节点

| 字段 | 必填 | 说明 |
|---|---|---|
| 状态 | ✓ | 无问题 / 不严重 / 严重影响沟通 |
| 具体表现 | 当状态≠无问题时必填 | 如 "n/l 不分" / "前后鼻音不分" / 等 |

> **方法论依据**：90-95% 中国学员发音不影响沟通，例外（n/l 不分等）需 First Priority。

#### 语法支撑（听口共用）

| 字段 | 必填 | 说明 |
|---|---|---|
| 状态 | ✓ | 到位 / 不到位 / 严重缺失 |
| 判断依据 | ✓ | 中翻英语法表现 + 学员对复杂句的自述 |

> **方法论依据**：语法支撑是听力阶段 b 的核心卡点，也是口语句型表达的基础。MCES 模型本身不含语法，但语法是诊断必查项。

#### 训练前置依赖

| 字段 | 必填 | 说明 |
|---|---|---|
| 前置依赖列表 | ✓ | 多条/一条/无 |

**规则：**
- 若发音"严重影响沟通" → **First Priority**，先搞发音再积累语料
- 若语法"严重缺失" → 视译训练前必须先补**句子成分专项**
- 若都正常 → 无前置依赖，直接按方案训练

> **下游 Solution 设计的硬约束**：训练前置依赖**不是 nice-to-have，是 gating 条件**。`barry-solution` 拿到档案后必须先消化前置依赖（在训练菜单的 Week 1-2 内排专项），再铺常规训练菜单。前置依赖未解决就铺训练 = 学员卡在前置项无法前进。

### Section 4 · Solution

由 `barry-solution` Skill 产出/更新，并基于此生成专属训练 Skill。

| 字段 | 必填 | 说明 |
|---|---|---|
| 训练目标 | ✓ | 3 个月内可达，对齐 Section 1 学习目标 |
| 训练前置依赖（继承 Section 3） | ✓ | 若有，方案前 X 周专攻前置项 |
| Four Strands 配比 | ✓ | Meaning-focused input / output、Language-focused、Fluency development 各占比 |
| 每日训练菜单 | ✓ | 具体到任务和时长，对应 Section 3 卡点 |
| 训练素材类型 | ✓ | 听力素材、口语任务、词汇/搭配储备等 |
| 阶段里程碑 | ✓ | 1 周 / 1 月 / 3 月 各自检查点 |

## 档案写入约定（共用条款，所有 SKILL 必须遵守）

> 这一节是工程层面的硬约定，避免多个 SKILL 各自写文件时撞坏档案。

### 文件结构约定

档案统一用 H2 heading 分 section，**严格按顺序排列**：

```markdown
# Barry 英文学习档案

## Section 1 · 个人基本情况
...
> 更新于 YYYY-MM-DD by barry-profile

## Section 2 · 目前水平
...
> 更新于 YYYY-MM-DD by barry-assessment

## Section 3 · 卡点
...
> 更新于 YYYY-MM-DD by barry-assessment

## Section 4 · Solution
...
> 更新于 YYYY-MM-DD by barry-solution
```

**关键规则：**
- Section heading 必须严格匹配正则 `^## Section [1-4] · `
- Section 之间用空行分隔
- 每个 section 末尾必须有 `> 更新于 YYYY-MM-DD by barry-<skill>` 标记

### 写入流程（必须按此顺序）

每个 SKILL 写入档案时，必须执行以下 5 步：

```
Step 1 · 检查 + 备份
  - 如果文件存在: cp profile.md profile.md.bak.<timestamp>
  - 如果文件不存在: 跳过备份, 走"创建新文件"分支

Step 2 · 读全文
  - 一次读完整 profile.md 到内存
  - 若 Read 工具返回截断警告(文件超过默认 2000 行),改用 offset/limit 分段读完整文件再合并到内存

Step 3 · 解析 + 替换
  - 按 ## Section N · 严格匹配 anchor
  - 如果目标 section 已存在 → 替换该 section 内容（保留前后 section）
  - 如果目标 section 不存在 → 按 1→2→3→4 顺序插入到正确位置

Step 4 · 整文件覆写
  - 用 Write 工具一次性覆写完整文件（不要用 Edit 局部改）
  - 写入失败 → 立即从 .bak 恢复

Step 5 · 验证 + 清理
  - 验证写入后所有 section heading 正则仍能匹配
  - 验证写入前已有的 section 内容没被破坏
  - 验证通过 → 删除超过 7 天的 .bak 文件
```

### 反模式（禁止）

- ❌ 用 Edit 工具局部改 section（容易撞坏 anchor）
- ❌ 不备份直接覆写
- ❌ 用模糊匹配定位 section（如只匹配"Section 2"忽略 ·）
- ❌ 跨 section 操作（一个 SKILL 只动自己负责的 section）
- ❌ 改 section heading 格式

### 冲突处理

如果 Section 4 已存在但 Section 2/3 被更新：

> "档案 Section 2/3 已变化（你的水平/卡点更新了）。Section 4 训练方案是基于旧的 Section 2/3 生成的，要不要重新生成？输入 `/barry-solution` 重新出方案。"

## 训练日志（独立文件，不进档案 Section）

> **路径**：`~/.barry-english/training-log.md`
>
> **设计原则**：档案是"画像"（学员是谁 / 在哪 / 卡在哪 / 计划是什么）；训练日志是"流水"（每天做了什么）。两者职责分离，不混在一个文件。

### 写入触发

每次专属训练 Skill 完成一次训练后，**自动追加一条**到 `~/.barry-english/training-log.md`。

### 推荐字段

```markdown
## YYYY-MM-DD（周 X）

- **时长**：实际训练 X 分钟（计划 Y 分钟）
- **完成训练类型**：[Strand × 方法]，如 "MFI 听力复述 / LFL 中→英视译 / FD 4-3-2 加速"
- **当日产出词块数**：N 个（已同步到 Quiz 词库）
- **学员主观感受**（可选）：
- **异常 / 触发信号**（可选）：如 "今天发音特别卡" / "感觉听力进步了" 等 —— 这些是档案过期的早期信号
```

### 与 Section 4 的关系

- **Section 4** = 训练**计划**（应该做什么）
- **training-log.md** = 训练**记录**（实际做了什么）
- 长期 log 可用于 review 计划与实际的 gap，未来给 AI 做 plan adjustment 提供依据

### 与 Quiz Web 的关系

- 训练日志只记"产出多少词块"
- 词块本身存到 `fancy-vocab.md`（统一词库）
- Quiz Web live read `fancy-vocab.md`（不读 training-log.md）

## 与 Barry 方法论的对齐说明

本 schema 的诊断坐标系**不直接照搬 MCES 节点**，而是按方法论的实际诊断框架：

| 模块 | 诊断坐标系 | MCES 模型对应 |
|---|---|---|
| 听力卡点 | 两步链路（阶段 a S→E / 阶段 b E→C/M） | 描述链路过程，不作为诊断坐标 |
| 发音 | S 节点状态 | 三档：无问题 / 不严重 / 严重影响沟通 |
| 语法 | 独立判断（专项 vs 当固定搭配背） | MCES 不直接覆盖 |
| 口语水平 | CEFR 级别（来自中翻英） | 综合表现的级别评估，不细分维度 |

**MCES 是过程模型，不是诊断坐标系。** MCES 描述链路（口语 M→C→E→S 渐渐过渡到 M→S 直连；听力 S→E→C→M 渐渐过渡到 S→M 直连），用于解释训练原理；诊断时按实际可观察的卡点维度走。

**口语"被动 vs 主动"是普遍状态描述（所有中国学员都是被动多主动少），不是 per-学员诊断字段，所以不进 schema。**训练目标"被动→主动"反映在 Section 4 的训练菜单设计里。
