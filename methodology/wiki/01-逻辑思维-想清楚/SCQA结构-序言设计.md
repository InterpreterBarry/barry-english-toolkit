# SCQA 结构(序言设计)

> 开场白/序言的四步结构:**S**ituation / **C**omplication / **Q**uestion / **A**nswer。
> 在"想清楚"板块中,是 [[Phase2-金字塔结构|Phase 2]] HOW 章节的**落地工具之一**——金字塔塔尖在开场时展开的方式。**含 3 个变体**(开门见山式 / 突出焦虑式 / 突出信心式)。

## 核心

**SCQA = 让听众自己「问出」你想回答的问题。** 不是你抛结论,是你**铺好路**,让听众脑子里自动产生那个问题,你再把答案(= 金字塔塔尖)给出去。

## WHAT:四个要素

| 要素 | 含义 | 一句话 |
|------|------|-------|
| **S**(Situation) | 背景 | 与主题相关的**公认事实** |
| **C**(Complication) | 冲突 | 背景之下产生的**困难或复杂性** |
| **Q**(Question) | 疑问 | 听众脑海里**自然产生的问题** |
| **A**(Answer) | 回答 | 金字塔塔尖——**你的答案** |

```
S  ────→   C   ────→   Q   ────→   A
公认事实     出问题了     你要回答啥    我来答
(建立共识)  (激发关注)   (对齐焦点)   (亮出结论)
```

## WHY:为什么是 SCQA 而不是直接抛结论

### 直接抛结论的问题

- 听众**没进入语境**,结论像从天而降,接不住
- 没有「为什么这事重要」的铺垫,听众觉得**和我无关**
- 没有共识基础,听众可能**当场反驳前提**

### SCQA 的解法

- **S 建共识**:先说大家都同意的事 → 听众点头,愿意往下听
- **C 制造张力**:公认事实之下出了什么问题 → 听众注意力被抓住
- **Q 锁焦**:所以我们要解决什么 → 听众的问题和你的问题**对齐**
- **A 给答案**:金字塔塔尖 → 听众已经**想听了**,你才给

**本质**:**先激活听众的问题,再回答**,不是反过来。

## 标准版:S → C → Q → A(通用推荐)

```
S:公司过去三年营收稳定增长,团队翻了三倍。     (公认事实)
C:但从今年 Q2 开始,会议时间占了总工时的 40%,    (冲突出现)
   项目延期率从 10% 升到 35%。
Q:我们的效率问题到底卡在哪?                      (问题浮现)
A:今天我想说的是——不是人手不够,             (塔尖给出)
   是会议管理失控。
```

## 三种变体(根据场景调整顺序)

| 变体 | 顺序 | 适用 |
|------|------|------|
| **开门见山式** | A → S → C | 决策层汇报、时间紧张、观众已知背景 |
| **突出焦虑式** | C → S → A | 需要先抓眼球、重大问题警示 |
| **突出信心式** | Q → S → C → A | 论坛演讲、需要共鸣、把听众脑中已有疑问拉出来 |

### 示例对比

**标准版(S-C-Q-A)** · 一般汇报:
> 我们过去三年稳健增长(S)→ 但今年效率下降(C)→ 卡在哪(Q)→ 是会议管理(A)

**开门见山(A-S-C)** · 董事会汇报:
> 我建议砍掉 50% 会议(A)→ 背景是我们稳健增长(S)→ 但今年效率数据大幅下滑(C)

**突出焦虑(C-S-A)** · 内部警示:
> 效率掉到历史最低(C)→ 要知道我们过去三年都在增长(S)→ 今天我给一个答案(A)

**突出信心(Q-S-C-A)** · 大型演讲:
> 你有没有每周开会超过 20 小时(Q)→ 其实你并不孤独,大公司普遍如此(S)→ 数据显示 40% 工时在会议上(C)→ 今天我想说…(A)

## 4U 和 SCQA 的配合

**标题 = 塔尖的第一次亮相;SCQA 序言 = 塔尖的第二次亮相。**

```
[[4U原则-标题设计|标题(4U)]]          → 让他决定「进不进门」
    ↓
SCQA 序言                              → 让他决定「留不留下来」
    ↓
主体(金字塔结构)                     → 让他「被说服」
```

## HOW:写 SCQA 的三个检查

1. **S 是不是真共识?**——如果听众对 S 有异议,整个 SCQA 塌掉
2. **C 是不是真冲突?**——硬编的「冲突」听众感觉不到,就没张力
3. **Q 是不是听众会问的问题?**——不是你想回答的问题,是**听众脑子里自然产生**的问题

第三个最容易出错。写完 SCQA 后**默读一遍**,问自己:「到了 Q 这一步,听众真的会问这个吗?」如果觉得是硬塞进去的,回头改 S 或 C。

## SCQA 在金字塔中的位置

```
金字塔塔尖(A)
    │
    ▲ 从这里倒推回去,铺 S → C → Q
    │
听众的问题(Q)
    │
    ▲ 冲突制造这个问题
    │
冲突(C)
    │
    ▲ 建立在背景之上
    │
背景(S)
```

**本质**:SCQA 是金字塔塔尖的「开场包装」,**把听众从 S 一步步带到 A**。
