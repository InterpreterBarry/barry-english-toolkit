# 逻辑思维板块 · 总览(想清楚)

> 职场英文沟通方法论的**第一环**:**先想清楚,再说清楚**。**2-Phase 主干** + 通过 Mental Models 持续提升思考质量。**Barry 的基本立场**:英文说不出来,往往不是英语问题,是**中文都没想清楚**。

## 板块定位

```
英文沟通方法论
├── 底层认知(地基)
└── 三大支撑能力(平行)
    ├── 翻译能力(译清楚)
    ├── 公众演讲(说清楚)
    └── 逻辑思维(想清楚)         ← 本板块
```

## 板块入口:有效沟通 + 逻辑

在进入 2-Phase 主干之前,先把基本概念定义清楚。

### 什么是有效沟通

| 层级 | 定义 |
|---|---|
| **基础** | **倾听者能够复述你所说的话**——最好是能使用你的逻辑 |
| **高阶** | **理解 + 认同**你的观点 |

### 如何实现有效沟通

| 层面 | 内容 |
|---|---|
| **理性层面** | **逻辑思维**(本板块的全部) |
| **感性层面** | 营造良好的情绪环境(本板块不展开,见 [[0-公众演讲板块总览|公众演讲]] / [[Mental-Models-总览|Mental Models]] 沟通类) |

### 什么是逻辑 ⭐

> **逻辑就是对信息的分类以及信息传递的顺序,且这种分类和顺序符合受众认知常识。**

这是 Barry 课程里逻辑的**一句话定义**——记住"**分类 + 顺序 + 符合受众认知**"三个要点。

## 整体框架:2-Phase 主干

```
Phase 1  明确核心问题
         ├── 1.1 找真问题(5 Whys + 谬误识别)
         ├── 1.2 对齐定义
         └── 1.3 对齐标准
              ↓
              ↓ Mental Models 持续提升思考质量
              ↓
Phase 2  金字塔结构
         ├── WHY:为什么学逻辑
         ├── WHAT:金字塔长什么样(纵向 + 横向)
         └── HOW:怎么搭出来(4U + SCQA + 主体设计)
```

**核心设计理念**:
- **Phase 1**:思考核心问题,**实战中三个子动作同时进行**
- **Phase 2**:用金字塔结构拆解 + 表达
- **没有 Phase 3 / 4 质检** —— 实战中"用得不多",合成口诀(原"质检四问")已删除
- **想提升整体思考质量** → **积累 Mental Models**(详见下方"持续提升路径")

## WHY:为什么学逻辑

> 逻辑思维不是"用来考试的理论",它是一个**实用工具**——直接影响你在职场里能不能被理解、被记住、被认同。

### 1. 应用场景极广

逻辑表达覆盖职场几乎所有高频场景:**演讲与主持 / 工作部署 / 书面汇报 / 问题解决**。只要你需要"让别人听懂你在说什么",**逻辑就在起作用**。

### 2. 提升沟通效率

**(a) 易于受众记忆**——科学家发现,人类大脑**无法在短时间内快速记住 7 项以上的要点**。超过 5 项就该用金字塔结构分组归类。逻辑结构不是让你显得专业,**而是真实地降低听众的认知负担**。

**(b) 易于受众理解和认同**——**与受众达成共识的逻辑结构**更容易获得理解和认同,**大大降低沟通成本**。不是你说得有多对,而是对方能不能跟上你的思路。

详见 [[Phase2-金字塔结构]] 的 WHY 章节(展开案例 + A/B 对照练习)。

## 持续提升路径:积累 Mental Models ⭐

整个方法论里**有一条独立的提升路径**——

> **想提升整体思考质量,就要尽可能多地积累 Mental Models。**

```
积累 Mental Models  ──→  在 Phase 1 / Phase 2 各阶段调用
            ↓
   思考质量持续上一个台阶
```

### 哪里用过 Mental Models

[[Phase1.1-找真问题]] 已经用了两个:
- [[第一性原理-First-Principles|第一性原理]]——拆到底层事实,从零重组
- [[奥卡姆剃刀-Occam-Razor|奥卡姆剃刀]]——多个解释都能说通时,先验证最简单的

### Mental Models 板块入口

→ [[Mental-Models-总览|Mental Models 板块总览]]

板块里 Barry **会持续放新的 Mental Models 进去**——决策思考类(第一性原理 / 奥卡姆剃刀 / 二阶效应 / 反转思维)、批判思维类(Sagan-Standard / Hitchens-Razor 等)、沟通类(WHAT-So-What-Now-What 等)。

> **课后作业**:学员自己搜集、整理 Mental Models,嵌入到对应 Phase。模型越多,各阶段思考质量越高。

## HOW:完整链路怎么跑

```
1. Phase 1.1:用 5 Whys 往下挖,5 个谬误排除伪问题
   → 调用第一性原理验证根因
   → 调用奥卡姆剃刀选最简单候选先验证

2. Phase 1.2:对齐定义,确保大家说的是同一件事

3. Phase 1.3:定"什么叫解决了"的衡量尺度

4. Phase 2.1 (WHY):明确为什么要搭逻辑结构

5. Phase 2.2 (WHAT):金字塔结构展开
   → 纵向问答(以上统下)
   → 横向归纳(时间/空间/重要性/MECE/比较利弊)
   → 横向演绎(三段论/问题分析/多米诺)

6. Phase 2.3 (HOW):落地工具
   → 4U 标题
   → SCQA 序言(+ 3 个变体)
   → 主体设计(自上而下 / 自下而上)
```

## 学员体感(完整链条走下来)

- **Phase 1**(1.1-1.3):**"原来我以前连问题都没搞清楚就开始干了"**(被打醒)
- **Phase 2**:**"原来想清楚之后,可以这么有结构地表达"**(学到工具)
- **Mental Models 持续积累**:**"原来这个框架可以越用越强,我可以不断往里加新工具"**(持续成长)
