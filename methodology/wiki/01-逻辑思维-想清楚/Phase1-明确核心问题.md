# Phase 1:明确核心问题

> 逻辑思维的**第一个 Phase**——把核心问题想明白。**实战中三个子动作同时进行**(找真问题 / 对齐定义 / 对齐标准),不是流水线。整个 Phase 1 完成后,才进入 [[Phase2-金字塔结构|Phase 2 金字塔结构]] 拆解。

## 核心理念

> **思考核心问题,然后直接进入框架去拆解就可以了。**

实战中**这三件事是一个连续的思考动作**,不是三个独立步骤——所以 Phase 1 是一个 Phase,内部三个子动作同时进行。

```
拿到问题
   ↓
[同时进行]
├── 1.1 找真问题:这是真问题吗?背后是什么?
├── 1.2 对齐定义:我们说的 X 具体指什么?
└── 1.3 对齐标准:怎样算解决了?
   ↓
进入 Phase 2 金字塔结构
```

## 三个子动作

### [[Phase1.1-找真问题|1.1 找真问题]]

> 工作中同事/老板抛过来的问题,**不一定是真问题**。

**基础工具**:
- **5 Whys**——连问 5 个"为什么",从表象找到根因
- **谬误识别**——5 个高频职场谬误识别"伪问题"

**质量放大器**(从 [[Mental-Models-总览|Mental Models 板块]] 调用):
- [[第一性原理-First-Principles|第一性原理]]——拆到底层事实,质疑假设
- [[奥卡姆剃刀-Occam-Razor|奥卡姆剃刀]]——多个原因都能说通时,先验证最简单的

详见 [[Phase1.1-找真问题]]。

### [[Phase1.2-对齐定义|1.2 对齐定义]]

> 例:会议吵了半天"用户体验不好"——A 说页面慢,B 说功能找不到,C 说客服响应慢。**三个人对"用户体验"的定义完全不同**。

**操作**:讨论之前花 1 分钟明确"我们今天说的 XX 具体指什么"。

详见 [[Phase1.2-对齐定义]]。

### [[Phase1.3-对齐标准|1.3 对齐标准]]

> 例:"要不要裁这个人?" → 标准是绩效?态度?潜力?团队配合?**标准不同,结论完全不同**。

**操作**:定义清楚后,追问"我们用什么标准判断这件事"。

详见 [[Phase1.3-对齐标准]]。

## 为什么这三件事要绑在一起

它们都是**"明确我们要思考什么"的不同侧面**——

| 子动作 | 防止什么 |
|---|---|
| 1.1 找真问题 | 防止解决**伪问题** |
| 1.2 对齐定义 | 防止跟别人**鸡同鸭讲** |
| 1.3 对齐标准 | 防止做了但被认为**没做到位** |

三件事**任何一件没做到位,后面的金字塔搭建都建立在沙地上**——结构再漂亮也没用。

## 实战中的优先级

并不是每一次都要严格走完三步——根据场景判断:

- **复杂业务问题 / 跨团队协作**:三步都要走,免得后面返工
- **简单决策**:可能只需 1.1(确认真问题)
- **熟悉团队 + 熟悉话题**:1.2 和 1.3 可能默认对齐,可以跳过

**判断逻辑**:**做这一步的成本 < 不做带来的返工成本**,就值得做。

## 完成 Phase 1 的标志

走完 Phase 1 之后,你应该能对自己说:
> "我清楚地知道**我要解决的问题是什么**(1.1)、**这个问题里的关键概念分别指什么**(1.2)、**怎样算解决了**(1.3)。现在可以开始用金字塔搭框架了。"

如果这三个问题任何一个还含糊,**不要急着进 Phase 2**——回头补。
