# PREP(Point — Reason — Example — Point)

> **观点表达的最干净结构**。先给观点 → 给原因 → 给例子 → 再回到观点。即兴演讲场景下临场就能套上。

## 板块定位

[[Mental-Models-总览|Mental Models]] / 演讲沟通类框架 / **即兴演讲框架(5 个)之一**。被 [[wiki/03-公众演讲-说清楚/即兴演讲/结构-5框架|公众演讲 / 即兴演讲]] 调用。

## WHAT:四步结构

| 字母 | 含义 | 一句话 |
|---|---|---|
| **P**oint | 观点 | 你的核心立场是什么 |
| **R**eason | 原因 | 为什么这个观点成立 |
| **E**xample | 例子 | 用具体案例支撑原因 |
| **P**oint | 观点(回扣) | 重申观点,首尾呼应 |

```
Point ────→ Reason ────→ Example ────→ Point
观点          原因          例子          观点
                                          (回扣)
```

## WHY:为什么 PREP 在即兴场景特别好用

**即兴演讲的最大风险是"讲着讲着跑偏"**——临场无脚本,容易绕远。PREP 通过**首尾的两个 Point** 把整段表达**钉死在观点上**:
- 第一个 Point 一开场就锚定方向
- 中间 Reason + Example 提供支撑
- 最后 Point 回扣 → 听众**记住的是观点本身**,不是细节

**对比标准演讲结构(Ramp → Roadmap → 3 PoDs → Q&A → Dessert)**:
- 标准结构适合**有准备**的演讲(5-15 分钟)
- PREP 适合**纯即兴**(30 秒到 2 分钟的临场表达)
- 复杂度差几个数量级

## HOW:操作示例

### 例 1:被问"职场新人最重要的能力是什么?"

| 步骤 | 内容 |
|---|---|
| **Point** | 我认为职场新人最重要的能力是**主动沟通** |
| **Reason** | 因为新人最大的痛点不是技能不足,是**信息不对称**——你不知道老板真正想要什么 |
| **Example** | 我刚入职那会儿做了一份汇报,自己觉得很完整,老板看完只问了一句"so what",我才发现我没回答他真正关心的问题 |
| **Point** | 所以**主动沟通**比埋头干活更重要——确认对方真正要什么,再开始干 |

### 例 2:被问"AI 时代还要不要学英语?"

| 步骤 | 内容 |
|---|---|
| **Point** | **要,而且更要学** |
| **Reason** | AI 能翻译,但**真实沟通是关于关系和判断**——这两件事 AI 替不了 |
| **Example** | 我去年跟一位英国客户开会,对方用了一个微妙的反讽,翻译软件直接翻成字面意思,我当场如果照着字面回应,合作就结束了 |
| **Point** | 所以**学英语不是为了翻译,是为了感知关系和场景**——AI 越强,这种能力反而越值钱 |

## 跟其他即兴框架的区别

| 框架 | 适合什么 | 跟 PREP 区别 |
|---|---|---|
| **PREP** | 表达观点 / 立场 | 强观点导向,首尾回扣 |
| [[What-So-What-Now-What]] | 反思 / 经验 | 偏向"事 → 意义 → 行动",不是观点导向 |
| [[Problem-Solution-Benefit]] | 提建议 / 解决问题 | 强解决方案导向,起点是"问题" |
| [[Comparison-Contrast-Conclusion]] | 比较类问题 | 多选项对比,起点是 A vs B |
| [[STAR-Situation-Task-Action-Result]] | 案例 / 经验回顾 | 叙事型,按时间顺序 |

## 常见误区

| 误区 | 真相 |
|---|---|
| "PREP 很简单,直接讲就行" | 不练就用不好——临场容易**跳过 Reason 直接进 Example**,变成"我觉得 X,比如 Y" |
| "PREP 只能讲一个观点" | 也可以套两轮 PREP——P-R-E-P → 转折 → P-R-E-P,但**临场最好别超过两轮**,会乱 |
| "Reason 和 Example 区别不大" | 有区别——Reason 是**逻辑论证**(为什么),Example 是**事实支撑**(具体到什么人/事/场景) |
