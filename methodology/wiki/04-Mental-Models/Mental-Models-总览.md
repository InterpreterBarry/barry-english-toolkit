# Mental Models 总览

> Barry 长期积累的思维模型素材库。**按功能分两大类:思维/思考类、演讲沟通类框架**。不是百科式收藏,而是真正能在职场英文沟通场景里调用的工具。**Barry 会持续更新新的 Mental Models 进来**。

## ⭐ 这板块跟其他板块的关系

Mental Models 板块**横跨服务**两个支撑能力:

```
[[0-逻辑思维板块总览|逻辑思维板块]]  ←──  调用思维/思考类 Mental Models
                                    ↘
                                     提升思考质量

[[wiki/03-公众演讲-说清楚/0-公众演讲板块总览|公众演讲板块]] ←──  调用演讲沟通类框架
                                    ↘
                                     可直接套用的说话框架
```

**核心立场**:模型**不是用来炫技的**,是用来**压缩决策成本 / 表达成本**的。同一个场景,套对模型 → 省 90% 的临场思考。

## 整体架构(两大类)

```
Mental Models
│
├── 思维/思考类 ⭐
│   ├── 决策思考(4 个):提升思考质量,被逻辑思维板块调用
│   │   ├── 第一性原理 / 奥卡姆剃刀 / 二阶效应 / 反转思维
│   │   └── (持续添加:沉没成本 / 机会成本 / 概率思维 等)
│   └── 批判思维(4 个):判断别人主张是否成立的筛子
│       └── Sagan-Standard / Hitchens-Razor / Alder-Razor / RISA-Framework
│
└── 演讲沟通类框架 ⭐
    ├── 日常沟通框架(7 个):场景化说话框架
    │   └── WSW-NW / WHAT-敬酒词 / PSB / ADD / 4I / AAA / CRAC
    └── 即兴演讲框架(5 个):公众演讲板块的纯即兴结构口诀
        └── WSW-NW / PREP / PSB / CCC / STAR
```

> **WSW-NW 和 PSB 既是日常沟通框架,也是即兴演讲框架**——同一个 Mental Model 在两个场景下都成立,**不重复定义**。

## 一、思维/思考类

**定位**:**不是临场说话框架**,是"想清楚"过程中用来**提升思考质量**的工具。学员**积累更多 → 调用更多 → 思考质量持续上一个台阶**。

### 1.1 决策思考(4 个 · 被逻辑思维板块调用)

| 模型 | 一句话 | 在逻辑思维里典型应用场景 |
|---|---|---|
| [[第一性原理-First-Principles]] | 拆到最底层事实,质疑假设,从零重组 | [[Phase1.1-找真问题|Phase 1.1]]:验证 5 Whys 挖出的根因是事实还是假设 |
| [[奥卡姆剃刀-Occam-Razor]] | 多个解释都说得通,先验证最简单的 | [[Phase1.1-找真问题|Phase 1.1]]:多个候选根因时,选最简单的先验证 |
| [[二阶效应-Second-order-Effects]] | 方案的直接结果会引发什么连锁反应? | 思考方案时追问"然后呢?",避免短期对长期错 |
| [[反转思维-Inversion]] | 先想怎样一定会失败,然后避开那些坑 | 验证方案前主动想"怎样会搞砸",列搞砸清单对照检查 |

**会持续更新进来**:沉没成本 / 机会成本 / 风险偏好 / 概率思维 等。

### 1.2 批判思维(4 个 · 判断别人主张是否成立的筛子)

**定位**:判断**别人的主张**是否成立、**自己要不要接招**。来源:Pursuit of Wonder + Bo Seo(世界辩论冠军)。

| 模型 | 用途 |
|---|---|
| [[Sagan-Standard]] | 主张越颠覆,证据门槛越高 |
| [[Hitchens-Razor]] | 无证据的主张,可以无证据驳回 |
| [[Alder-Razor]] | 无法实验/观察验证的东西,不值得争论 |
| [[RISA-Framework]] | 卷入争论前先过 4 个问题:真?重?具体?目标一致? |

**关键认知**:这 4 把剃刀**不是杠精工具**,是让你**省出时间和心力**去讨论真正重要的事情。

## 二、演讲沟通类框架

**定位**:**临场可直接套用的说话/演讲框架**——每个对应一个具体场景。被 [[wiki/03-公众演讲-说清楚/0-公众演讲板块总览|公众演讲板块]]、沟通三要素 等场景调用。

### 2.1 日常沟通框架(7 个 · 现场可直接套用)

| 模型 | 场景 | 一句话 |
|---|---|---|
| [[What-So-What-Now-What]] | 日常寒暄 / 反思 / **即兴演讲** | 咋了 → 那咋了 → 然后呢 |
| [[WHAT-敬酒词框架]] | 敬酒 / 致辞 / 介绍 | 背景 → 关系 → 故事 → 感谢 |
| [[Problem-Solution-Benefit]] | 说服 / 销售 / 提建议 / **即兴演讲** | 抛痛点 → 给方案 → 讲好处 |
| [[ADD-问答模型]] | 面试 / Q&A / 被质问 | 直答 → 举例 → 关联对方 |
| [[4I-反馈模型]] | 给下属/同事反馈 | 事实 → 影响 → 建议 → 后果 |
| [[AAA-道歉模型]] | 冒犯 / 失误 / 误解 | 认错 → 共情 → 改进 |
| [[CRAC-辩论模型]] | 据理力争 | 结论 → 规则 → 分析 → 结论 |

### 2.2 即兴演讲框架(5 个 · 公众演讲板块调用)

公众演讲板块的**即兴演讲(Impromptu)结构**5 个口诀框架——给临场表达提供"骨架":

| 模型 | 结构 | 适用 |
|---|---|---|
| [[What-So-What-Now-What]] | 咋了 → 那咋了 → 然后呢 | 反思类 / 经验类即兴 |
| [[PREP-Point-Reason-Example-Point]] | Point — Reason — Example — Point | 表达观点 / 立场 |
| [[Problem-Solution-Benefit]] | Problem — Solution — Benefit | 提建议 / 解决问题类 |
| [[Comparison-Contrast-Conclusion]] | A 的特点 — B 的特点 — 综合结论 | 比较类问题 |
| [[STAR-Situation-Task-Action-Result]] | Situation — Task — Action — Result | 描述案例 / 经验回顾 |

> **WSW-NW 和 PSB 在两类里都出现**——它们既是日常沟通框架,也是即兴演讲框架。**同一个 Mental Model,两个场景调用**。

## 如何选模型

Barry 的经验是**按场景反查**,不是按模型正查:

```
我现在要做决策 / 推理对吗     → 思维/思考类 / 决策思考(第一性原理 等)
对方在断言 / 我要不要信       → 思维/思考类 / 批判思维(四剃刀)

我现在要 说服                 → PSB
我现在要 反馈                 → 4I
我现在要 道歉                 → AAA
我现在要 据理力争             → CRAC
有人在质问我                  → ADD
寒暄没话聊                   → WSW-NW
敬酒 / 介绍                   → WHAT-敬酒词

被叫上台即兴讲                → 5 个即兴框架(WSW-NW / PREP / PSB / CCC / STAR)
```
