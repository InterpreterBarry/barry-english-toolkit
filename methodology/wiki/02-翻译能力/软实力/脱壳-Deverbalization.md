# 脱壳(Deverbalization)

> 把 Message 从原语言的文字外壳里剥离出来,再用目标语言重新包装。**中到英翻译的核心动作**。

## 核心理念

> **信息/意思是灵魂,语言只是肉身。**
> 翻译不是翻文字,是翻 Message。

- 理想:地道中文 → 地道英文
- 现实:**直接过不去**
- 所以:必须**脱壳**——先从中文里把 Message 剥离出来,再用英文重新包装

## WHAT:脱壳 = 一个动作

**关键认知**:脱壳**只有一个动作**——把文字背后的 Message 找出来。

这个动作**对所有人都一样**。不同的是脱壳之后用什么词来重新表达——**这才是因人而异的**。

```
中文原文 → [脱壳] → 赤裸的 Message → [重新表达] → 英文
         (对所有人一样)              (因人而异)
```

## WHY:为什么要脱壳

**中文和英文是两种不同密度的语言**:
- 英文像**铁块**:精准的词就够了
- 中文像**棉花**:需要很多字来表达同一意思

如果直接字对字翻译,会产生:
- **翻译腔**(机械对应,不自然)
- **逻辑链断裂**(中文隐含的逻辑没显化)
- **文化违和**(中文表达习惯直接嫁接到英文)

脱壳就是**把文字从意思中解放出来**,让意思本身成为翻译的对象。

## HOW:脱壳的操作

### 示例 1(Barry 经典案例)

**原文**:供应商和生产线的数据相关性比较薄弱

**Step 1 · 脱壳**(所有人一样):**两边的数据对不上**

**Step 2 · 重新表达**(因人而异):
- 基础水平:*The data from A and B don't match.*
- 较高水平:*The two teams are not using the same data metrics.*

## 脱壳在翻译链路中的位置

脱壳是 [[中到英翻译-三步走]] 的第二步:

```
[[切分]](找出句子里有几个核心意思)
    ↓

    ↓
[[逻辑连词显化]](显性标出中文隐含逻辑)
```

## 脱壳做不到位的典型症状

- 英到中译文**翻译腔重**(例:"如果我看这里,我看到你完成了英语的这个本科学位")
- 中到英译文**中式英语**(例:字对字直译成语/俗语)
- 听起来像**机器翻译**而不是"人话"

修复:回到脱壳阶段,问自己"**他到底想说什么**",忽略他具体用了什么字。
