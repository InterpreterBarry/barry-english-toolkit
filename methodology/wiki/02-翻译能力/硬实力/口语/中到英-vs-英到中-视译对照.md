# 中到英 vs 英到中:视译方向对照

> 视译有两个方向,**对称、不可互替**。常见失败是只做一个方向,导致输入输出失衡。

## 核心定位

视译训练有两个方向,都跑 [[MCES模型]] 链路,只是方向相反:

```
中到英视译:M → C → E    (输出方向)
英到中视译:E → C → M    (输入方向)
```

中文(C)始终是中介,英文(E)和意思(M)在两端互换位置。

## WHAT:对照表

| 维度 | 中到英视译 | 英到中视译 |
|---|---|---|
| **解决的问题** | 口语说不出(被动→主动词汇) | 听完就忘(意思构建慢) |
| **训练方向** | 输出 | 输入 |
| **操作** | 看中文,口头说英文 | 看英文,口头说中文 |
| **MCES 链路** | M → C → E → S | S → E → C → M |
| **目标产出** | 准确表达 + 逐步摆脱中文 | 快速 + 意思不丢 |
| **详细页** | [[视译训练-Sight-Interpreting]] | [[文字到意思-英到中视译]] |
| **配套训练** | [[中到英翻译-三步走]] | [[声音到文字-听写四步]] |
| **Four Strands 归属** | [[Meaning-focused-Output]] | [[Meaning-focused-Input]] |

## WHY:为什么必须双向

只做一个方向 = 单边训练 = 系统性失衡:

- **只做中到英** → 口语进步,但听力不上去 → 听不懂客户/老板说什么
- **只做英到中** → 听力进步,但开口还是卡 → 能听懂但表达不出

Barry 体系强调 [[输入输出并行原则]]——视译训练这一项就要**双向并行**,不是输出做多了听力就自然好。

## HOW:训练顺序

### 同一段材料,**先输入后输出**

按这个顺序走,**一段材料的学习价值被榨干**,效率远高于"四段材料各做一种训练":

1. [[声音到文字-听写四步]](听清)→ 同段
2. **英到中视译**(听懂,加速 E→C→M)→ 同段
3. **中到英视译**(说出来,打通 M→C→E→S)→ 同段

### 时间分配

如果你只有 **30 分钟/天**:
- 15 分钟英到中(输入)+ 15 分钟中到英(输出)
- **不要 30 分钟全押一个方向**

## 常见误区

| 误区 | 问题 |
|---|---|
| 只练自己弱的方向 | 真实场景是双向沟通,单方向练不出综合能力 |
| 中到英 vs 英到中交替不同段做 | 应该**同一段材料走两个方向**,而不是不同段交替 |
| 觉得"翻译路径"是弱点要避免 | 翻译路径是必要桥梁,加速到极致后自然跳过 |

## 示例:30 分钟训练流程

材料:一段 ~150 词的职场英文(如客户邮件)

| 时间 | 步骤 | 训练目标 |
|---|---|---|
| 0-5 分钟 | 听写一遍 | 听清每个词 |
| 5-15 分钟 | 英到中视译,同段重复 3 次,速度递增 | 加速 E→C→M |
| 15-25 分钟 | 中到英视译(用同样素材的中文版) | 主动词汇调动 |
| 25-30 分钟 | 同段中到英重做一遍,逐轮提速 | 推动 M→S 直连(流利度训练) |

**关键**:四道工序用**同一段材料**,信息在大脑里反复加工,不同方向多次曝光。
