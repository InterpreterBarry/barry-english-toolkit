# Fluency Development

> Four Strands 的第四条线:**把已知内容用得更快、更流畅,不学新东西**。**没有时间压力就不算 fluency activity**。

## WHAT:速度焦点的自动化训练

Fluency Development(流利度发展)是四条线里**唯一独立维度**的那条——它不是"学新东西",而是把已经学过的东西**从"组装模式"切换到"调用模式"**。

典型活动:速读、4/3/2 技巧、repeated reading、限时写作、听已经熟悉的简单故事、同一 Mini Presentation 的逐轮提速。

## WHY:为什么这条线独立存在

语言使用时,大脑有两种模式:

- **组装模式**:一个词一个词拼出句子,走 M → C → E(详见 [[MCES模型]])
- **调用模式**:整块取用,直接走 M → S 直连

**组装模式有上限**——再熟练的组装都没有直接调用快。要达到"英语思维",必须让**常用表达进入调用状态**。

Fluency Development 的作用就是:**强制大脑切换到调用模式**。没有这条线,学习者就算词汇语法都懂,说起来依然卡顿。

## HOW:四个前提条件(Nation)

1. **全部内容已知**——无生词、无新语法
2. **注意力在传达 / 接收意思**(不是在形式上)
3. **有时间压力**——Nation 原话:**没有压力就不算 fluency activity**
4. **大量输入或输出**

第 3 条是分界线。Retelling 说得完整仔细 ≠ fluency;**同一段内容,第二遍要求更快说完**,这才进入 fluency。

## HOW:时间压力的物理机制

时间压力的作用不是"逼学员焦虑",而是**剥夺组装模式的可能性**:

- 没时间压力 → 学员有空余 CPU 做 M→C→E→S(想中文 → 翻成英文 → 检查语法 → 开口)
- 加时间压力 → CPU 不够了,大脑**自动选择最有把握的现成表达**

这正是从翻译思维过渡到英语思维的**物理机制**,也是为什么 Nation 的 4/3/2 实验里**不仅速度提升,语法准确性和表达复杂度也同步提升**——因为学员被压力逼着**放弃那些"勉强能拼出来"的表达,只用"闭着眼都能说"的那部分**。

## HOW:Barry 体系的落地

**Retelling 加入逐轮提速**:
- 第一遍不限时,追求完整 → [[Meaning-focused-Output]]
- 第二遍起时间减半 → Fluency Development
- 同一任务覆盖两条线

**Mini Presentation 加入逐轮提速**:
- 参考 [[4-3-2-技巧]]:4 分钟 → 3 分钟 → 2 分钟
- 同一段话讲三遍,不是换题讲三遍

### 注意事项

- 前提是**已经完成第一遍学习**(材料全部已知)——不能直接上限时
- 时间压力要**适度**,目的是"推一把",不是制造焦虑
- 不纠正语法,只追求流畅度

## 示例

**是 Fluency Development**:
- 酒吧里朋友第二遍讲同一个故事(Calum 的例子)
- 同一篇演讲,分别讲 4 / 3 / 2 分钟
- 已熟悉内容的重复听力、重复阅读

**不是 Fluency Development**:
- 第一次讲一个新话题(还在 meaning-focused output)
- 不限时的 Retelling(没压力就不算)
- 听全新内容(不满足"全部已知")
