# 翻译能力 · 板块总览(译清楚)

> 三大支撑能力之一(逻辑思维 / 翻译能力 / 公众演讲),平行关系。**翻译能力 = 中↔英的转换能力**——不只是字面对应,更是把意思在两种语言之间打通。

## 命名逻辑:为什么叫"翻译能力"

**不只是字面 word-for-word translation**——更广义的"把意思在两种语言之间打通":
- 抓文字背后的意思
- 用另一种语言**地道**重新表达

**为什么不叫"语言能力"**:语言能力太宽泛,容易被误解为"逻辑能力"或"表达能力"。**用"翻译"或"转换"措辞,体现"准确性"**——核心是**中↔英的转换**。

## 硬实力 vs 软实力 ⭐

**关键:硬/软不是按"能力领域"切,而是按"关注焦点"切**——同一个训练动作(如视译),关注焦点不同 → 训练的是不同实力。

| | 副标题(更准确的命名) | 关心什么 | 在 [[MCES模型]] 上 |
|---|---|---|---|
| **硬实力** | **= 能调用储备** | 听不听得懂 / 说不说得出来 —— 语言储备能不能调用 | M-C-E-S 各节点 + 节点间连接 |
| **软实力** | **= 调不出来时换路径** | 调不出来时,**怎么换路径**仍然把意思传过去 | 跨节点的灵活替换 |

**为什么命名要加副标题**:原命名"硬实力 = 听力口语能力 / 软实力 = 翻译技巧"会误导——"翻译技巧"覆盖了听力 + 口语,所以"听力口语能力"和"翻译技巧"不是互斥的。**副标题"能调用储备 / 调不出来时换路径"才真正反映两者的本质差异**——是关注焦点不同,不是能力领域不同。

### 关键观点

**软实力 = 弥补硬实力跟 native 之间永远存在的差距**。

无论硬实力练到多高,作为非母语者,跟 native 永远有缝隙。**软实力是填这条缝的工具**——硬实力的**永远伴侣**。

## 学员的硬/软实力可以不平衡

经典例子:**A2 水平学员英文破碎但能正常工作**——硬实力低 + 软实力高。

| 状态 | 评价 |
|---|---|
| 好处 | 在实战沟通中已经具备脱壳思维 |
| 坏处 | 不精进硬实力 → 无法**精准 / 独立 / 一针见血**地表达 |

**Barry 的训练建议**:
- 大部分时间先提升硬实力
- 提到一定程度再着重练软实力
- **两者分开训练**——硬实力靠视译类,软实力靠脱壳类

## 这板块的命题

| 板块 | 命题 |
|---|---|
| 逻辑思维 | 怎么**想清楚** |
| 翻译能力 | 怎么把想清楚的内容**译清楚**(中↔英转换) |
| 公众演讲 | 怎么把译清楚的内容**说清楚** |

三大板块**平行**——都不创造 Message,但分别处理"思考 / 转换 / 传达"。
