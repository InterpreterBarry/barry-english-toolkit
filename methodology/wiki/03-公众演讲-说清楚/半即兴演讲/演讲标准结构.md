# 演讲标准结构

> 经典演讲架构:**Ramp → Roadmap → 3 PoDs → Q&A → Dessert**。把 [[Communication-Outline]] 想明白的内容,装进这个容器里。

## 板块定位

[[0-公众演讲板块总览|公众演讲板块]] / [[wiki/03-公众演讲-说清楚/半即兴演讲|半即兴演讲]] / **内容(Content)章节**。

## WHAT:完整结构

```
Opening   → Ramp(开场钩子)
          → Roadmap(路线图)

Body      → 3 PoDs(三个 Points of Discovery)
          → Q&A(中段或结尾互动)

Closing   → Dessert(情感甜点)
```

## 各部分作用一览

| 部分 | 作用 | 要点 |
|------|------|------|
| **Ramp** | 开头抓注意力 | **7 秒定生死**。技巧:More "You" / Sexy number / Ask a question / Make a confession / "Imagine..." / Story |
| **Roadmap** | 告诉听众接下来的安排 | 时长 / 结构 / 互动规则 |
| **3 PoDs** | 主体内容(Points of Discovery) | **不超过 3 个**要点;每个要点结束做 summary |
| **Q&A** | 与听众互动建立信任 | **永远不要以 Q&A 结尾**,你需要强有力的收尾 |
| **Dessert** | 制造情绪,让人记住 | 用 Story / Metaphor / Image;最好**首尾呼应 Ramp** |

## WHY:为什么是这个顺序

- **前 7 秒决定听众是否关注 → 必须有 Ramp**
- **没有 Roadmap → 听众焦虑,不知道还有多久 / 讲啥**
- **3 PoDs 是人脑记忆上限**(见 [[4U原则-标题设计]] 背后的认知心理学:3 是黄金数字)
- **Q&A 不能结尾** → 因为 Q&A 的能量起伏不可控,你要把能量最高点留给自己
- **Dessert 首尾呼应 → 形成回路**,听众会感受到"完整性"

## HOW:六个 Ramp 技巧

1. **More "You"**:"Every one of you has felt this..." —— 直接跟听众对话
2. **Sexy number**:"87% of our projects missed deadline last year" —— 冲击性数字
3. **Ask a question**:"Raise your hand if you've..." —— 强制互动,眼神聚焦
4. **Make a confession**:"Three years ago, I made the worst call of my career" —— 示弱拉近距离
5. **"Imagine..."**:"Imagine you walk into the boardroom and..." —— 进入场景
6. **Story**:一个 30 秒的真实故事 —— 情感连接

## HOW:Dessert 的形式

- Story / Anecdote —— 回到 Ramp 开的故事,首尾呼应
- Metaphor —— "This is not a finish line. It's a new starting line."
- Image —— 留一个画面在听众脑中
- 核心信念一句话 —— "We're not selling software. We're buying back people's time."

**Dessert 的底层心理**:**峰终效应**——听众记忆里最深的两个点是"高峰"和"结尾"。Dessert 是为了打造强结尾。

## HOW:4 项 Techniques(让内容打动人的技巧)⭐

来自 Feishu 课件最新版,**4 个让 PoDs 内容打动人的具体技巧**:

| 技巧 | 说明 |
|---|---|
| **Stories** | 用真实故事代替抽象概念——人脑天然记忆故事而不是数据 |
| **Metaphors** | 用熟悉的事物类比陌生概念——"AI 是新的水电"比"AI 是基础设施"更易懂 |
| **Active language** | 用主动语态、动作动词——"我们决定砍掉 50% 会议"比"50% 会议被砍掉了"有力量 |
| **Refrain**(重复) | 关键句重复 3 次——让听众记住:"It's not about us. It's not about them. It's about all of us." |

**这 4 项跟 Ramp 6 技巧的区别**:Ramp 技巧是**开场抓注意力**,Techniques 4 项是**整段演讲内容的"装饰品"**——可以贯穿在 PoDs 和 Dessert 里。

## 3 PoDs 的内容怎么来

**来自 [[Phase2-金字塔结构]]**:

想清楚板块已经把内容拆成**金字塔**(一个主论点 + 三个支撑论点)。演讲阶段把这个金字塔**压缩为 3 个 PoDs**:

```
Phase 3 金字塔                    Body 3 PoDs
  主论点(Key Message)      →     演讲的核心信息
  ├─ 支撑 1                →     PoD 1
  ├─ 支撑 2                →     PoD 2
  └─ 支撑 3                →     PoD 3
```

[[SCQA结构-序言设计]] 提供了 PoDs 之前的 **Ramp 段落**中的铺垫顺序(Situation → Complication → Question → Answer)。

## 示例:完整结构

**主题**:说服团队换项目管理工具

- **Ramp**(Sexy number):"去年我们开了 340 个 project meetings。猜猜其中有多少是**纯粹用来对齐状态**的? 210 个。"
- **Roadmap**:"接下来 10 分钟我想说清楚三件事。然后留 5 分钟 Q&A,最后我会给一个承诺。"
- **PoD 1**:旧工具的 3 个痛点(数据支撑)
- **PoD 2**:新工具的 3 个优势(演示支撑)
- **PoD 3**:迁移方案(时间线 + 责任人)
- **Q&A**:5 分钟
- **Dessert**(回到 Ramp 数字):"我们不是在换工具,我们是在**把那 210 场会议拿回来**。那是每人每周 4 小时 —— 足够我们做更重要的事。"
