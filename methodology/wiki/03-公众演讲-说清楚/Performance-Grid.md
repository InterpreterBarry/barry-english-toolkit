# Performance Grid(演讲表现矩阵)

> **内容 × 表现**的二维四象限。用来解释:为什么好内容讲不好会失败,以及演讲的目标状态是什么。

## WHAT:四象限

|  | 内容强 | 内容弱 |
|---|---|---|
| **表现好(engaging)** | **Inspired**(目标状态) | **Entertained**(好看没干货) |
| **表现平(flat)** | **Frustrated**(有料但听不进去) | **Apathetic**(全方位无感) |

### 四象限听众反应

- **Inspired**:被激励,愿意行动 —— 演讲的目标
- **Entertained/Amused**:开心但觉得肤浅 —— 脱口秀陷阱
- **Frustrated/Bored**:感兴趣但难以集中注意力 —— **中国职场最常见的翻车区**
- **Apathetic/Angry**:走神或假装在听 —— 最糟

## WHY:这张图为什么重要

**核心认知**:**光有好内容不够,还要会讲**。

很多技术背景的 Speaker 都掉在 **Frustrated** 这一格:内容很硬、数据很扎实,但**听众接不住、记不住、不愿意行动**。原因不是内容不行,是表现形式不匹配。

这张图把公众演讲的价值讲得很清楚:

- 内容 = [[0-逻辑思维板块总览]] 负责(想清楚)
- 表现 = 公众演讲板块负责(说清楚)
- 两个都拉到"强",才是 Inspired

## HOW:诊断自己演讲处在哪个象限

演讲结束后三个问题自问:

1. **听众能复述我的核心信息吗?** 不能 → 内容弱(回 Content 补)
2. **听众过程中有反应(点头、笑、记笔记)吗?** 没有 → 表现弱(回 Delivery 补)
3. **听众结束后有行动吗?** 没有 → 说明至少有一维掉了

## 反例:典型 Frustrated 现场

职场汇报经典场景:

- PPT 写满字,Speaker 照着念
- 数据和逻辑都对,但没有 [[演讲标准结构|Opening]] 钩子
- 也没有 [[Communication-Outline|Relevance]] 跟听众建立连接
- 结束时观众礼貌鼓掌,**走出会议室什么都不记得**

→ 这就是典型 Frustrated:内容在,表现不在。

## 目标象限:Inspired

Inspired 不是"讲得好玩"那么简单,它有三个特征:

1. 听众**理解了**核心信息(Focus 达成)
2. 听众**在乎**这个信息(Relevance 达成)
3. 听众**愿意行动或转变**(Do 达成)

所以 Inspired = Content 方法论 + Delivery 方法论 + Mentality 方法论**全部到位**。
