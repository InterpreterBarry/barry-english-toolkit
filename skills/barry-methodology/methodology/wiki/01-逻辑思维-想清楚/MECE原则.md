# MECE 原则(相互独立、完全穷尽)

> **M**utually **E**xclusive, **C**ollectively **E**xhaustive。金字塔每一层的**检验尺**——分类要「不重不漏」:**分清** + **分净**。

## 核心

**MECE 不是搭金字塔的起点,是金字塔搭完之后的「体检工具」。** 你归纳完、演绎完,每一层过一遍 MECE,看看有没有漏、有没有重。

## WHAT:两个原则

| 原则 | 含义 | 要求 |
|------|------|------|
| **相互独立**(ME) | 划分在同一维度上有明确区分 | **分清**——类别之间不重叠 |
| **完全穷尽**(CE) | 分解全面、周密、完整、无遗漏 | **分净**——合起来覆盖全部 |

```
相互独立:A ∩ B = ∅           (没有交叉)
完全穷尽:A ∪ B ∪ C = 全集     (没有漏掉)
```

## WHY:不 MECE 会怎样?

- **重叠**:论点互相盖,听众觉得「这不是在重复吗?」 → 说服力打折
- **遗漏**:关键维度没覆盖,听众能举反例 → 直接戳穿
- **混维度**:同一层同时按「性别」和「是否管理者」两个维度切,本该交叉成 2×2 四象限,却被平铺成「男 / 女 / 管理者 / 非管理者」并列 → 互相重叠,逻辑发虚

**底层原因**:人类大脑自动往「互斥 + 穷尽」的方向找安全感。不 MECE 的结构,听众即使说不出哪里不对,也会**直觉上不信**。

## HOW:MECE 的核心技巧——**一层只用一个标准**

这是整个 MECE 最实用的操作规则:

> **每一层只选用一个标准进行切分,不混。**

### 错误示例

同时按「性别」和「是否管理者」划分:

```
男员工 / 女员工 / 管理者 / 非管理者 ← 4 类,但男员工里也有管理者,重叠!
```

### 正确做法

- **第一层**:先按「性别」切:男员工 / 女员工
- **第二层**(如需要):每个类目下再按「是否管理者」切

```
男员工
├─ 男管理者
└─ 男非管理者
女员工
├─ 女管理者
└─ 女非管理者
```

这样**每一层都单一维度**,互斥 + 穷尽都保证了。

## 应用场景:从下往上 vs 从上往下

| 搭建方向 | 场景 | MECE 侧重 |
|---------|------|-----------|
| **从下往上** | 例会工作汇报 | **重点保证「不重叠」**(宁可少列不重复) |
| **从上往下** | 战略决策、咨询 | **严格 MECE**(两个都要) |

- **例会**:你主要目的是信息同步,漏了一小块不致命,但重了会让人觉得啰嗦
- **战略**:漏一个维度可能就亏钱,所以必须两个都保

## 示例:拆解「销售额下降」原因

**不 MECE 的糟糕拆法**:

- 广告投放不足
- 竞争对手降价
- 产品质量问题
- 北区业绩差 ← 这条和上面三条不是一个维度

**MECE 的拆法**(第一层按「商业三要素」切):

- **流量**(广告投放 / 渠道结构)
- **转化**(产品/销售/客服)
- **客单价**(定价 / 产品组合)

三者互斥、穷尽——**销售额 = 流量 × 转化率 × 客单价**,数学上就闭环了。然后每一类下面再往下拆。
