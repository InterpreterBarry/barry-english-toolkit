# Phase 1.1:找真问题

> 工作中同事或老板抛过来的问题,不一定是"真问题"。**先判断这背后真正要解决的是什么,再开始干活。** [[Phase1-明确核心问题|Phase 1]] 的第一个子动作。

## 核心

**伪问题的代价**:把别人抛来的话当成真问题,你就开始解一道错题——答得再漂亮也是白费。Phase 1.1 的全部工作,就是**先分清真问题和伪问题**。

## WHAT:两个基础工具 + Mental Models 调用

```
Phase 1.1 找真问题
├─ 基础工具一:5 Whys(往下挖根因)
├─ 基础工具二:谬误识别(排除伪问题)
└─ 调用 [[Mental-Models-总览|Mental Models]] 提升思考质量:
   ├─ [[第一性原理-First-Principles]](验证根因是事实还是假设)
   └─ [[奥卡姆剃刀-Occam-Razor]](多个根因选最简单的先试)
```

> Mental Models **不是 Phase 1.1 的子步骤**,而是从外部 Mental Models 库**调用**进来的"质量放大器"。学员可以**持续积累更多 Mental Models**,嵌入这里来提升思考质量(详见 [[Mental-Models-总览]])。

## 基础工具一:5 Whys(五个为什么)

连问 5 个"为什么",从表象找到根因。(源自丰田生产方式,丰田佐吉提出、大野耐一推广。)

> 老板说:"团队执行力太差。"
> → 为什么执行力差?→ 任务经常延期
> → 为什么延期?→ 大家不清楚优先级
> → 为什么不清楚?→ 每次开会没有明确产出
> → 为什么没有产出?→ 会议没有议程和决策机制
> → **根因:不是执行力问题,是会议管理问题**

5 Whys 的局限:它只保证你**往下挖**,不保证你挖到的是事实——这也是为什么需要调用 [[第一性原理-First-Principles|第一性原理]] 来做验证。

## 基础工具二:5 个高频职场谬误

用来排除伪问题的"思考卫生"清单:

| 谬误 | 一句话 | 识别场景 |
|---|---|---|
| **相关 ≠ 因果** | 数据相关不代表有因果 | "销量下降是因为换了包装"——真的吗?还是竞品降价了? |
| **滑坡谬误** | A → B → C → 灾难,中间跳了多少步? | "如果我们降价,品牌就会崩"——推理链成立吗? |
| **虚假二选一** | 看起来只有两个选项,实际还有第三条路 | "要么裁人,要么亏损"——真的只有这两条? |
| **以偏概全** | 用个例推全局 | "我朋友用了很有效"——一个人不是样本 |
| **充分 vs 必要混淆** | "够不够"和"要不要"分不清 | "有好产品就一定能成功"——好产品是必要条件,不是充分条件 |

**用法**:拿到一个问题先扫一遍这 5 条,看对方的陈述里藏着哪个坑。

## 调用 Mental Models 提升质量

### 调用 [[第一性原理-First-Principles|第一性原理]]

**核心**:把问题拆到不能再拆,找到最基本的事实,再从这些事实重新构建。

**与 5 Whys 的配合**:5 Whys 帮你找根因(从表象往下挖),第一性原理帮你**验证根因**(这个根因是事实还是假设?)。

> 老板说"我们要招更多人"
> → 5 Whys 找到根因:项目总延期
> → 第一性原理追问:项目延期真的是因为人不够吗?
> → 拆解事实:每周有效工时只有 60%,40% 花在无效会议上
> → **真正要解决的不是招人,是会议效率**

### 调用 [[奥卡姆剃刀-Occam-Razor|奥卡姆剃刀]]

**核心**:如果两个解释都能说通,选更简单的那个。

**用法**:5 Whys 挖出多个可能的根因 → 奥卡姆剃刀帮你**先验证最简单、最直接的那个**。

> 这个月销售额下降了 15%
> → 候选根因 A:大环境 + 竞品降价 + 消费者偏好变化(复杂)
> → 候选根因 B:上个月涨价了 10%(简单)
> → 奥卡姆剃刀:先验证 B,如果 B 能解释,就不需要 A

## HOW:Phase 1.1 完整操作

```
拿到问题
   │
   ▼
5 Whys 往下挖(工具一)
   │
   ▼
谬误扫描(工具二) ──→ 是伪问题? ──→ 回去跟对方对齐
   │
   ▼ 看起来是真问题
   │
调用第一性原理:这个根因是事实还是假设?
   │
   ▼
多个候选根因? ──→ 调用奥卡姆剃刀:先验证最简单的
   │
   ▼
进入 [[Phase1.2-对齐定义|1.2 对齐定义]]
```

## 想做得更好?积累更多 Mental Models

> 越多 Mental Models 嵌入到 Phase 1.1,**找真问题的能力越强**。

详见 [[Mental-Models-总览]]——里面有持续更新的 Mental Models 库,你可以挑适合"找真问题"场景的(决策思考类、批判思维类)嵌入进来。
