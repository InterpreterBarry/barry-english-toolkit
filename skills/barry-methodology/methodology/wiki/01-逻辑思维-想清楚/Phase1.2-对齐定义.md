# Phase 1.2:对齐定义(我们在讨论什么?)

> 明确了真问题之后,把**准确的定义**给出来,确保所有人说的是同一件事。

## 核心

**同一个词,每个人脑子里的画面不一样。** 不先对齐定义,接下来的讨论都是浪费时间——大家看似在吵一个问题,其实在吵三个问题。

## WHAT:Phase 1.2 要做的一件事

在讨论之前,花 1 分钟明确:**"我们今天说的 XX,具体指的是什么?"**

## WHY:不定义的后果

讨论听起来很激烈,结论却没人能执行——因为每个人都按自己理解的定义在往下推。

## 示例:会议上吵"用户体验不好"

```
A 说的是"页面加载慢"   ── 技术性能问题
B 说的是"功能找不到"   ── 信息架构问题
C 说的是"客服响应慢"   ── 服务流程问题
```

三个人对"用户体验"的定义完全不同——**不先对齐,谁也说服不了谁**。对齐后会发现:今天这个会只能解决其中一条,另外两条各自需要不同的负责人和资源。

## HOW:三步对齐

1. **把抽象词列出来**:哪些词大家"看起来都懂,其实各懂各的"?(用户体验、效率、ROI、增长、创新……)
2. **每个人先写自己的定义**:不讨论先动笔,避免群体思维压掉差异。
3. **合并成一个版本**:这场讨论里,XX 具体指什么、不指什么。

## 和 Phase 1.1(找真问题)的区别

- **Phase 1.1**:"你说的这个问题,是不是真问题?"(挑战问题本身)
- **Phase 1.2**:"你说的这个词,具体是什么意思?"(统一口径)

两个子动作对应两类失败:Phase 1.1 失败 → 解错题;Phase 1.2 失败 → 各解各的题。
