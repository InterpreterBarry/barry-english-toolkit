## 2.1 WHY:为什么用金字塔(为什么学逻辑)

> 逻辑思维不是"用来考试的理论",它是一个**实用工具**——直接影响你在职场里能不能被理解、被记住、被认同。

### 1. 应用场景极广

逻辑表达覆盖职场几乎所有高频场景:
- **演讲与主持**
- **工作部署**
- **书面汇报**
- **问题解决**

只要你需要"让别人听懂你在说什么",**逻辑就在起作用**。

### 2. 提升沟通效率

#### (a) 易于受众记忆 ⭐

科学家发现,**人类大脑无法在短时间内快速记住 7 项以上的要点**。超过 5 项就该用金字塔结构,**对要点进行分组整理、归类概括**。

逻辑结构不是让你显得专业——**而是真实地降低听众的认知负担,帮他们记住你**。

#### (b) 易于受众理解和认同

**与受众达成共识的逻辑结构**更容易获得理解和认同,**大大降低沟通成本**。

不是你说得有多对,而是**对方能不能跟上你的思路**。逻辑好的人,**不需要重复解释,也不会被追问"你到底想说什么"**。

### A vs B 信息传递接龙(经典练习)⭐

同样的信息,**有没有逻辑结构,差别有多大?**

#### A 版本(无逻辑结构)

> 今天我为大家推荐一些健康的食物,希望有助于改善大家的饮食结构。先推荐**葡萄**,因为葡萄富含维生素 C,食用后既可以提高免疫力,又可以缓解疲劳。接下来大家可以多吃**土豆**,属于蔬菜,因为土豆含有淀粉,吃了可以增加力气。大家也可以选择**奶酪和酸奶**,它们属于奶制品,可以为身体补充钙质。另外一种蔬菜也很推荐,也就是**萝卜**,萝卜能消食化积、通气清热,俗话说:"冬天萝卜赛人参。"另外建议大家每天吃一个**苹果或橘子**,因为它们富含维生素 C,可以缓解疲劳并且提高免疫力。当然,**牛奶和黄油**也是非常好的补钙食物。

> 读完,30 秒后:你能复述出几种食物?说出几个原因?

#### B 版本(有逻辑结构:餐前 → 餐中 → 餐后)

> 接下来,我为大家推荐 **9 种**健康食物,希望能改善各位的饮食结构。我会分别从**餐前、餐中和餐后**为大家介绍。
>
> **餐前**,建议大家多吃水果,因为水果富含维生素 C,可以帮助我们缓解疲劳、提高免疫力。我将**按个头从大到小**推荐:**苹果、橘子和葡萄**。
>
> **餐中**,建议大家多吃蔬菜。我**按形状**来推荐:圆的可以选择**土豆**(含有淀粉,可以增加力气);长的可以选择**萝卜**(消食化积、通气清热,俗话说"冬天萝卜赛人参")。
>
> **餐后**,推荐多服用**奶制品**补钙(餐后饮用有利于人体钙的吸收)。**液体**奶制品:**牛奶、酸奶**;**固体**奶制品:**黄油、奶酪**。

#### 核心 insight ⭐

> **两段文字包含完全相同的信息,食物种类一模一样。差别只在于有没有逻辑结构。**
>
> 这就是逻辑的力量——**不是让信息变多,而是让信息变得可传递、可记忆**。

| 版本 | 体感 |
|---|---|
| A 版本 | 信息散装,随机顺序 → 读完什么都记不住 |
| B 版本 | 三层结构,逻辑清晰 → 读完能复述框架 |

## 2.2 WHAT:金字塔结构长什么样

### 金字塔的两个维度

```
                   ┌──────────┐
                   │   结论   │     纵向 = 问答关系
                   └────┬─────┘     (以上统下)
              ┌─────────┼─────────┐
              ▼         ▼         ▼
           ┌─────┐   ┌─────┐   ┌─────┐
           │论据A│ ← │论据B│ ← │论据C│ 横向 = 归纳 or 演绎
           └─────┘   └─────┘   └─────┘
              │         │         │
            ...        ...       ...    (继续拆,直到够具体)
```

### 纵向关系:[[金字塔-纵向问答关系|以上统下 / 问答关系]]

每一层都在回答上一层的 **What / How / Why**。
- 为什么会这样?(Why)
- 怎样才能这样?(How)
- 为什么这样说?(还是 Why)

通过纵向问答,你引导文章形成一种**疑问-问答式的对话**,使读者**带着兴趣**了解你的思路进展。

详见 [[金字塔-纵向问答关系]]。

### 横向关系一:[[金字塔-横向归纳逻辑|归纳]](从多个具体信息归纳出结论)

| 归纳方式 | 说明 |
|---|---|
| **时间逻辑** | 第一步 → 第二步 → 第三步(按绝对时间 / 业务流程顺序) |
| **空间逻辑** | 北区→东区→南区→西区(按现实/虚拟空间或公式拆解,如 3C / 4P / QCD / 5W2H / SWOT) |
| **重要性逻辑** | 最重要的放第一个,**找决胜点,一击命中** |
| **MECE 原则** | 不重不漏(详见 [[MECE原则]]) |
| **比较利弊** | 方案 A 利/弊 vs 方案 B 利/弊 → 综合 → 结论 |

详见 [[金字塔-横向归纳逻辑]]。

### 横向关系二:[[金字塔-横向演绎逻辑|演绎]](从前提推导出结论)

| 演绎模式 | 结构 |
|---|---|
| **三段论** | 大前提 → 小前提 → 结论 |
| **问题分析(根-解-损)** | 问题 → 原因 → 方案 → **损害评估** |
| **多米诺效应** | A→B→C→D(连锁反应,推理性强) |

详见 [[金字塔-横向演绎逻辑]]。

### 横向归纳的两类总结技巧

| 类型 | 技巧 |
|---|---|
| **描述类思想** | 找**共性** |
| **行动类思想** | 找**共同的行动结果** |

## 2.3 HOW:怎么把金字塔搭出来(三件套落地工具)

### 1. 标题设计:[[4U原则-标题设计|4U 原则]] ⭐

| U | 含义 | 解释 |
|---|---|---|
| **Usefulness** | 实用性 | 受众通过标题就能知道**看完文章后可以获得什么** |
| **Uniqueness** | 独特性 | 通过标题告诉受众**为什么要读这篇文章 / 文章有何特殊之处** |
| **Urgency** | 紧迫感 | 通过标题告诉受众**为什么现在就得去阅读**,塑造紧迫感与危机感 |
| **Use slogan** | 口号式 | 把标题设计成**口号样式**,唤起受众的责任心和使命感 |

详见 [[4U原则-标题设计]]。

### 2. 序言/开场白设计:[[SCQA结构-序言设计|SCQA 结构]] ⭐(含 3 个变体)

#### 基础结构:S → C → Q → A

| 字母 | 含义 |
|---|---|
| **S**ituation | **背景**——与主题相关的公认事实(时间/地点/情境) |
| **C**omplication | **冲突**——在这个背景下,产生了什么困难 |
| **Q**uestion | **问题**——这种困难让读者脑海中产生什么疑问 |
| **A**nswer | **答案**——你对这个问题做出的解答(就是金字塔的塔尖) |

#### 三个变体(根据沟通目的选)

| 变体 | 顺序 | 何时使用 |
|---|---|---|
| **开门见山式** | A → S → C | 想以**结论吸引受众** |
| **突出焦虑式** | C → S → A | 想**强化矛盾**,以冲突引起注意 |
| **突出信心式** | Q → S → C → A | 想**突出解决方案**,以答案使受众信服 |

详见 [[SCQA结构-序言设计]]。

### 3. 主体设计:自上而下 vs 自下而上

#### 自下而上(整理大量信息)

适用场景:**面对一堆碎片素材,但还没有确定的观点**(如分析、调研、例会汇报)。

```
1. 列出所有要点
2. 找出各要点之间的逻辑关系(归类分组 + 横向逻辑)
3. 得出结论(概括)
```

#### 自上而下(已有明确论点)

适用场景:**已经知道要说什么,需要展现说服力**(如汇报、邮件、年度业绩、晋升答辩)。

**前提**:对业务情况比较熟悉。

```
1. 设计标题(确定主题/中心思想)
2. 分析受众的需求和关注点
3. 设计序言(SCQA)
4. 确定要点
5. 组织支持上层观点的内容和材料
6. 重复 4-5
7. 设计横向逻辑
```
