# 金字塔 · 横向演绎逻辑

> 金字塔横向维度之二:**从前提推导出结论**。像**侦探破案**——一环扣一环,每一步都推得动下一步。

## 核心

**演绎 = 由前到后。** 从一个已经立住的大前提出发,一步步推到结论。

## WHAT:演绎的两种标准结构

| 演绎模式 | 结构 | 举例 |
|---------|------|------|
| **三段论** | 大前提 → 小前提 → 结论 | 所有迟到都扣钱 → 小王今天迟到了 → 小王要扣钱 |
| **多米诺逻辑** | A → B → C → D | 原材料涨价 → 成本上升 → 利润压缩 → 必须调价 |

## WHY:演绎适合「先结论后信息」

**当你已经有明确结论**并想说服别人时,用演绎:

- 年底业绩汇报:我要说我这一年干得漂亮 → 大前提+小前提 → 结论
- 晋升答辩:我应该被晋升 → 拆分论据一层层推下来
- 邮件/汇报:有结论就先抛,再用演绎往下支撑


## HOW:搭演绎推理时问自己两个问题

### 1. 大前提立得住吗?

"所有迟到都扣钱" 是真的吗?(公司明文规定,还是你的假设?)

很多演绎链失败,失败点都在**大前提是未经验证的假设**——这时候回到 Phase 1.1 用 [[第一性原理-First-Principles|第一性原理]] 验证。

### 2. 推理链中间有没有跳步?

**多米诺逻辑最容易出的坑**:从原材料涨价跳到必须调价,中间的「利润是否真的被压缩到不能内部消化」这一步是不是默认省略了?

这就是 [[Phase1.1-找真问题|滑坡谬误]] 的典型表现——中间跳了多少步,你没数。

## 归纳 vs 演绎 · 实战中混用

**真实汇报的结构**通常是**两者串联**:

```
开头用 SCQA 抛结论(演绎角度)
    ↓
用归纳给出证据(数据/案例/市场分布)
    ↓
用演绎推到方案(大前提 → 小前提 → 结论)
    ↓
回到塔尖总结
```
