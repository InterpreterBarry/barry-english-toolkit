# 语块概念(Meaning Unit)

> **No word is an island.** 没有一个单词是孤岛。职场英文的基本单位不是"单词",是**语块**——**核心两类:固定搭配 + 固定句型**。
>
> 注:"Meaning Unit(意义单位)"是 Barry 给这个概念起的教学化叫法,学界对应的成熟术语是 **formulaic sequences / lexical chunks**(程式化序列 / 词块);"语块作为一个整体存取、绕开实时组装"的认知负荷论证,可上溯到 Pawley & Syder (1983) 对 native-like 流利度的经典分析。

## 核心观点 / WHAT

**"意思对意思",不是"字对字"。**

- 错误学法:把"提出" = put out、"想法" = idea 分开记,拼起来用
- 正确学法:"提出想法" = **put forward an idea**(作为一个整体记忆、一个整体调用)

## 语块的两类核心 ⭐

Barry 方法论里的"中英语块",**核心就两类**:

### 一、固定搭配(Collocations)

| 中文 | 英文搭配 |
|---|---|
| 百忙之中 | take time out of your busy schedule |
| 推进入职流程 | move forward with the onboarding process |
| 跳出舒适区 | step out of your comfort zone |
| 提出想法 | put forward an idea |
| 重排优先级 | reprioritize the projects |
| 跟进事项 | follow up on something |

**特征**:动词 + 介词 + 名词的**整体表达**,**有固定搭配**,改一个词就不地道。

### 二、固定句型(Sentence Patterns)

| 中文 | 英文句型 |
|---|---|
| 相比于 A 我更喜欢 B | I prefer B to A |
| 只是因为...并不意味着 | Just because... doesn't mean... |
| 刚...就... | No sooner had I... than... |
| 一边...一边... | While... |
| 我宁愿做 A 而不是 B | I would rather do A than do B |

**特征**:**整体框架**(语法结构),填进去具体内容就能用——但作为**整体**记,不要拆开分析每个词为什么这么排。

> **跟翻译能力/口语方法论的"训练目标两维度"对齐**:
> - 维度 1:单词(基础储备,抓本质含义)
> - 维度 2:**固定搭配 + 固定句型**(本页核心两类 = 这一维度的内容,抓背后意思;注:方法名"中英语块"涵盖单词 + 搭配句型两个维度,不单指这一维)
> 详见 [[中英语块-两个维度]]。

## WHY:为什么语块是一个整体(认知负荷视角)

为什么非得整块存?因为大脑在实时输出时,**没有余力做多步组装**:

```
单词记忆 → M → [组装:选词 + 搭配 + 时态 + 介词] → E    (慢、易错)
语块记忆 → M → [整块调用] → E                          (快、地道)
```

固定搭配作为**一个单元**存储,才能在实时沟通场景里被直接调用——这是"语块作为单位"的认知基础。

> "为什么要把语块放在单词前面优先积累""谁该语块优先(零基础 vs 有基础分界线)""默认策略对谁成立"——这些 **为什么优先 + 怎么落地** 的问题,见 [[语块优先原则]]。本页只负责"语块是什么"。

## 示例:一个语块如何拯救一整段表达

**Message**:希望对方重新排下这几个项目的优先级

**单词拼凑**:*Can you change the order... no... the priority... of these projects? Make some first, others later?*(卡、绕、不像人话)

**语块调用**:*Could you reprioritize these projects?*(一个词解决,且地道)

区别不在英语水平,在**有没有把 reprioritize 作为语块存下来**。
