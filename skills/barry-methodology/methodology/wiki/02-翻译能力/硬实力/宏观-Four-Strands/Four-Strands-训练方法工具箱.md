## 配比设计原则 ⭐

**只有 3 个 Strand 进入配比表**（MFI / MFO / LFL），三足鼎立均衡：

| 级别 | MFI | MFO | LFL |
|---|---|---|---|
| **A 级** | 1/3 | 1/3 | 1/3 |
| **B 级** | 1/3 | 1/3 | 1/3 |
| **C 级** | 1/3 | 1/3 | 1/3 |

> ⚠️ **核心警示**：中国学员普遍**过度倾向 LFL，忽略 MFI 和 MFO**。
> 设计训练方案时必须保证 MFI / MFO 的占比，**避免"三足鼎立"塌一边**。

### FD 不独立占配比（是嵌入策略）⭐

**FD 不是跟另外 3 个 Strand 同等重要的一条线**——它是融入复述(听材料=MFI;首轮产出=MFO;限时加压轮=FD)+ MFO Mini Presentation 的**时间压迫策略**：

| 级别 | FD 在 MFI/MFO 中的应用 |
|---|---|
| **A 级** | **完全不考虑**（基础阶段做不到，要求太高；A 级先打基础，B 级以上再加 FD，理由见 [[Four-Strands-与Barry方法论的融合]]）|
| **B 级** | 适当加入（AI 在复述 / Mini-Pre 训练时给"再压缩 X 秒"指令）|
| **C 级** | **一定要有**（每次复述 / 演讲都强制走 4-3-2 流程，时间逐轮压缩）|

> **设计落地**：barry-solution 设计 Section 4 训练菜单时，FD 不切独立时间块。B/C 级学员的训练菜单里，复述项和 Mini-Pre 项**直接标注"含 FD 加压"**（如 "Mini-Pre 8 分钟，含 4-3-2 加压"）。详见 4.1。

## Cross-Cutting 总原则（贯穿所有训练）

### 1. 词块积累必选

所有 AI 反馈（复述 / 演讲 / 中英互译 / 写作）**必须**包含：

- **词块（单词 + 固定表达 + 句型）积累** → 存学员本地专属词汇表
- 标准词汇表 **4 列格式**：英文 | 音标（**英 / 美**，按学员偏好）| 中文意思 | 英文例句 + 中文翻译
- 后续同步到 **Anki 单词卡网页**（详见 3.1）

### 2. 词块筛选门槛随学员级别单调递增 ⭐

**学员级别越高，入库门槛越高**。AI 在生成 Fancy Vocab 时按学员档案 Section 2 的综合 CEFR 决定筛选阈值——**跳过明显低于学员水平的基础词**：

| 学员级别 | 入库标准 | 不收的例子 |
|---|---|---|
| **A1 / A2** | 几乎所有不熟悉的词都收（被动词汇还在建立）| — |
| **B1** | 跳过最基础高频词，收次高频实词 + 关键搭配 | take / make / work / get 这些不收 |
| **B2** | 跳过 B1 通用词，收商务标准词 + 复杂搭配 | benefit from / indicate / suggest 这种通用词不收 |
| **C1** | 只收地道高阶词 + 修辞性搭配 | 任何 < C1 的基础或通用词都不收 |
| **C2** | 只收学术 / 政策抽象词 + 罕见地道表达 | 任何 < C2 的常见词都不收 |

> **设计意图**：避免 C1 学员词库里塞满 take / make 这种浪费时间的词，让 Fancy Vocab 始终在"略高于学员当前水平"的层次。

### 3. 训练频率 ⭐

- **核心原则**：尽量每天练，**不中断**
- **可休息**：每周最多 1-2 天，**不强制周末**（学员有的工作日忙周末空，有的反过来——按学员实际情况）
- **每天可轻量化**（疲劳时减时长不停训练）
- **允许 lightweight 重复检查**：训练 Skill 启动时检查"今天是否已训练过"，但不做硬阻拦（大多数学员每天只练一次）

### 4. 档案过期处理 ⭐

- ❌ **不主动定时提醒重测**（评估准确度还不够高，定时提醒会扰动学员）
- ✅ **学员主动汇报情况变化时触发**：
  - 学员说"我换工作了" / "我感觉水平进步了" / "我换学习目标了"
  - → AI 主动建议："那要不要**重新走一遍 onboarding 整个流程**？（profile + assessment + solution）"
- 重跑流程后 Section 1-4 全部刷新 + 专属训练 Skill 重生

## Strand 1 · Meaning-Focused Input（意义聚焦输入·MFI）

> **核心 purpose**：理解意思，焦点在内容不在形式（≠ 刻意学形式的 LFL；附带学习仍会发生）
> **方法论提醒**：中国学员的**最大缺口**（见 [[Four-Strands-与Barry方法论的融合]]）

### 1.1 分级阅读（Graded Reading）

| 维度 | 配置 |
|---|---|
| **素材来源** | **AI 生成**（网上现成分级读物多为故事/生活内容，职场相关少）|
| **询问学员** | "你喜欢读什么内容？"（默认推职场/业务相关）|
| **A/B/C 区分** | **长度 + 难度**（AI 按欧标 ABC 难度自己把控）|
| **AI 能做** | ✓ 生成内容 + 阅读理解跟进 |

### 1.2 分级听力 + 复述（Graded Listening + Retelling）

| 级别 | 素材来源 | 流程 |
|---|---|---|
| **A** | AI 推荐英语教学播客 / 网站（如 VOA Special English / Learning English / BBC Learning English）| AI 给链接 → 学员复述 → AI 打分 + 反馈 + 整理词块 |
| **B / C** | 学员上 YouTube / TED 自找，提供链接 | 直接复述（可用语音转文字 APP 如 Typeless / 闪电说作 backup）|

**复述方式**：偏好英文，中文也接受。
**流程参考**：`english-training` Skill 的 **step1-listening**——同样的复述 → 评分 → 反馈 → 词块积累流程。
**语音转文字 APP 用途**：长时复述 AI 语音功能可能宕机时的 backup；如 AI 工具本身支持长时间语音，**优先用 AI 直接听语音**。
**配套方法**：[[复述-Retelling]] / [[听力材料分级选择]]

## Strand 2 · Meaning-Focused Output（意义聚焦输出·MFO）

> **核心 purpose**：传达信息，焦点在内容不在形式（≠ 刻意学形式的 LFL；附带学习仍会发生）
> **方法论定位**：你的**强项**（Mini Presentation / 模拟会议已有）

### 2.1 对话 / 口语角（Conversation / Speaking Corner）

| 维度 | 配置 |
|---|---|
| **形态** | 利用 ChatGPT / Claude 的语音直接沟通能力，**AI = 外教**|
| **A/B/C** | 三级都必须练，主题难度按级别调整 |
| **AI 能做** | ✓ 实时语音对话 + 反馈 |
| **Backup** | AI 长时语音宕机 → 语音转文字 APP（同 1.2）|

### 2.2 Mini Presentation（迷你演讲）

| 维度 | 配置 |
|---|---|
| **流程** | AI 出题 → 学员 **1 分钟**构思 → **2 分钟**英文演讲 → AI 反馈 + 评分（1-5）|
| **话题来源** | 关联型（与当天听力/阅读相关延伸）/ 随机职场型（团队管理 / 沟通 / 决策 / 职业发展）|
| **评分维度** | Structure / Fluency / Vocabulary / Depth（综合打分，不分项）|
| **流程参考** | `english-training` Skill 的 **step2-impromptu** |
| **方法论依据** | [[Mini-Presentation训练方法论]] |
| **B/C 级特殊** | 加入时间压迫（4-3-2，见 4.1）|

### 2.3 写作（Writing）

| 维度 | 配置 |
|---|---|
| **触发** | AI 询问学员是否有每日写作需求（看个人偏好）|
| **反馈模型** | 语法层面 + 意思层面 + 词块 / 固定搭配积累（类似业务侧 Fancy Vocab）|
| **积累去向** | Markdown 词汇表 → 同步到 Anki 单词卡网页（见 3.1）|

## Strand 3 · Language-Focused Learning（形式聚焦学习·LFL）⭐ 方法最多

> **核心 purpose**：刻意关注语言形式 / 深度加工 / 间隔重复
> **方法论提醒**：Nation 立场是不应超过 25%，但 Barry 方法论判断中国学员可调整（被动→主动激活需要更多 LFL 时间，见 [[Four-Strands-与Barry方法论的融合]] 第 4 节）

### 3.1 Anki 单词卡（Anki Flashcards / Vocabulary Quiz Web）

#### 触发与基础配置

| 维度 | 配置 |
|---|---|
| **触发** | AI 主动询问用户要不要（预期大部分会同意）|
| **来源** | 每次训练都生成单词表（类似业务侧 Fancy Vocab）|
| **三类内容** | (a) 不熟悉的词 (b) 该级别应积累的 (c) **单词 + 固定表达 + 固定句型**（三类都要）|
| **4 列标准格式** | 英文 \| 音标（**询问美式 or 英式**，按学员偏好固定）\| 中文意思 \| 英文例句 + 中文翻译 |
| **本地存储** | 学员专属词汇表（本地 Markdown）→ 同步到 Anki 网页数据库 |
| **配套方法** | [[间隔重复-Spaced-Repetition]] / [[词汇学习-两种方式]] / [[中英语块-两个维度]] |

#### Anki 网页要复现的核心功能

##### A. Daily Vocabulary Quiz（每日 5 题）

- **题型**:英译中 **4 选 1**（从词库随机抽）
- **题面元素**:
  - 英文词条
  - 🔊 听单词按钮（TTS 或本地音频）
  - 英文例句（来自词库本身）
  - 4 个中文选项（A / B / C / D）
- **答题即时反馈**:
  - ✅ 正确 / ❌ 错误
  - 显示完整释义 + IPA 音标（**英式**,如 `/siːs ənd dɪˈsɪst ˈletə/`）
  - 🔊 重听音按钮
  - 💡 例句重展
  - 底部 "下一题 →" 按钮
- **进度计数**:X / 5（顶部）
- 5 题完成 → 今日 quiz done

##### B. 复习本（SM-2 间隔重复算法）

- **算法**:**SM-2（SuperMemo 2）**——经典间隔重复算法（Anki / SuperMemo 同源）
- **入库规则**:答错的题自动归入复习本
- **每张卡显示**:
  - 词条 + 间隔标签（如 `1d` / `2d`）
  - 中文释义
  - 例句
  - 元数据行:下次到期 / 间隔 / ease（SM-2 ease factor）/ 错误次数
- **操作**:"已掌握 ✓" 按钮（从复习本移除）
- **顶部 summary**:共 N 个 · 今日到期 N 个

##### C. 卡组管理

- 两类卡组:
  - 📖 **自动卡组**:每次 English Training session 自动生成 1 个卡组（按当天 session 主题命名,如 *"May 12 — Translation Standard Expressions (Para A: Cross-Team Coordination)"*）
  - 🎯 **自定义卡组**:学员可 "+ 新建卡组"
- 每个卡组卡片显示:
  - 标题
  - 张数 + 日期
  - "学习" 按钮（进入该卡组的 quiz 模式）
- 长期训练学员可积累 100+ 自动卡组（基于训练历史）

##### D. 🔄 更新题库按钮

- 本地词库（Markdown）变化后点击同步到 quiz 数据库
- 用于:每次新 session 产出新词后,学员手动触发同步

##### E. 解锁逻辑（可选）

- 学员可选:必须完成今日训练才能用 Quiz
- 设计意图:强制每日训练 + 即时复习闭环
- 默认不开启,作为 opt-in feature

#### 开源工程要点

| 项 | 实现建议 |
|---|---|
| 数据架构 | Markdown 为 single source of truth + localStorage 存 SRS 状态 |
| SM-2 算法 | 经典 SM-2 配置:`LEARNING_STEPS: [1, 10] 分钟 / GRADUATING_INTERVAL: 1 天 / DEFAULT_EASE: 2.5 / MIN_EASE: 1.3 / EASY_INTERVAL: 4 天 / EASY_BONUS: 1.3` |
| TTS 听音 | Web Speech API(浏览器原生)+ 服务端 TTS fallback |
| IPA 音标 | 来自词库(学员选英式/美式时已固定,默认英式)|
| 题库同步 | live read(server 每次直接解析 Fancy Vocab.md,不需要 sync 脚本)|
| 干扰项生成 | 4 选 1 的错误选项从同词库其他词随机抽 |

**词库 Markdown 格式约定:**

- Markdown 表格,header 至少包含 `Expression` / `Chinese` / `Example`(case insensitive);可选 `IPA` / `English Meaning`
- 列顺序灵活,用 colMap 映射
- 自动按 `en.toLowerCase()` 去重
- session 分组按 H3 heading `### Month Day, Year — Title`(如 `### May 12, 2026 — Translation Standard Expressions...`)
- 无日期 H3 视为"基础词汇集"

### 3.2 中→英翻译（Chinese to English Translation）

> 视译归 LFL（横跨 LFL/MFO，为简化归 LFL）。

| 级别 | 模式 |
|---|---|
| **A** | **笔译**（Written Translation）|
| **B / C** | **视译**（Sight Translation）|

| 维度 | 配置 |
|---|---|
| **难度匹配** | 句子长度 + 难度匹配 ABC 级别 |
| **流程** | AI 生成中文句/段（按级别）→ 学员翻译 → AI 打分 + 给出不地道表达 + 给好的版本 + 存 Fancy Vocab |
| **流程参考** | `english-training` Skill 的 **step3-translation** |
| **方法论依据** | [[中英语块加视译]] / [[视译训练-Sight-Interpreting]] / [[中到英-vs-英到中-视译对照]] |

### 3.3 英→中翻译（English to Chinese：听写 + 复述）

| 维度 | 配置 |
|---|---|
| **AI 能力边界** | ⚠️ AI 无法生成听力片段 → **学员自己找视频素材** |
| **AI 提供** | **听写四步法**的明确指导（直接引用 [[声音到文字-听写四步]]）|
| **听写重点** | A / B 级基础越薄弱越要打好听写基础（基础不打好后面学不上去）|
| **复述部分** | 见 1.2，不重复 |
| **方法论依据** | [[声音到文字-听写四步]] / [[文字到意思-英到中视译]] / [[听力问题诊断体系-Hearing-vs-Listening]] |

### 3.4 语法（Grammar）— 融入精读，不单独学

| 级别 | 内容 |
|---|---|
| **A** | 基础语法（简单句、并列句）|
| **B** | 进阶：主谓宾定状补 + 插入语 + 各种从句（主语 / 宾语 / 定语 / 状语从句）|
| **C** | 英文中最复杂的各类句子 |

| 维度 | 配置 |
|---|---|
| **方法** | 不单独学语法，**通过精读带出来**（见 3.5）—— 每天给一个英文句/短段落，做"英译中"翻译 + 拆分成分学语法 |
| **方法论依据** | [[语法-跨听口支撑]] |

### 3.5 精读（Intensive Reading）— 翻译三步走

参考 Barry **B 级快闪直播课**的"翻译三步走"：

1. **理解意思** → 翻成中文
2. **拆分成分** → 找主句、划分主谓宾
3. **切分意群 + 顺句驱动**

| 维度 | 配置 |
|---|---|
| **方法论依据** | [[文字到意思-英到中视译]] / [[顺句驱动]] |
| **关联** | 精读 = 语法的载体（见 3.4），不另外切时间 |

### 3.6 发音（Pronunciation：48 音标 / 连读 / 语调）

| 维度 | 配置 |
|---|---|
| **AI 能力边界** | ⚠️ AI 无法纠音 → **建议找 1V1 纠音老师** |
| **AI 提供** | **主动询问**学员发音是否有问题（筛"严重影响沟通"的 5-10%，详见 [[发音-S节点专题]]）|
| **方法论依据** | [[发音-S节点专题]] / [[Shadowing八步法]] |

## Strand 4 · Fluency Development（流利度发展·FD）

> **核心 purpose**：把已知内容用得更快更流畅 / 自动化
> **方法论定位**：**训练策略**，不占大比例；**A 级不需要**，B/C 级嵌入现有训练（不另起炉灶）

### 4.1 4-3-2 原则

| 级别 | 配置 |
|---|---|
| **A** | **不需要**（基础阶段做不到，要求太高；A 级先打基础，B 级以上再加 FD，理由见 [[Four-Strands-与Barry方法论的融合]]）|
| **B** | 适当加入 |
| **C** | 强化训练 |

| 维度 | 配置 |
|---|---|
| **做法** | 在 [[Mini-Presentation训练方法论\|Mini Presentation]] 和 [[复述-Retelling\|Retell]] 里**加入时间压迫感** |
| **关键前提** | 必须有 pressure to perform faster——**没时间压力 ≠ Fluency**（Nation 原话，见 [[Fluency-Development]] 条件 3）|
| **不另起炉灶** | 不切独立时间块，作为 B/C 级 MFO 训练的"加压方式"嵌入 |
| **方法论依据** | [[4-3-2-技巧]] |

## AI 能力边界一览（barry-solution 出方案时必须明确标注给学员）

| 训练方法 | AI 能做 | ⚠️ AI 不能做 / 需外部资源 |
|---|---|---|
| 分级阅读 | ✓ 生成内容 + 跟进理解 | — |
| 分级听力 + 复述 | A 级 ✓ 推素材 + 复述反馈 | B/C 级学员自找 YouTube/TED 素材 |
| 对话 / 口语角 | ✓ 语音对话（如平台支持）| 长时对话宕机 → 语音转文字 APP backup |
| Mini Presentation | ✓ 出题 + 反馈 + 评分 | — |
| 写作 | ✓ 反馈 + 词块积累 | — |
| Anki 单词卡 | ✓ 生成 + 同步本地词库 | 需要专门 Anki 网页（待 Barry 共享 dashboard 实现）|
| 中→英翻译（笔译/视译）| ✓ AI 出题 + 反馈 | — |
| 英→中翻译（听写 + 复述）| ✓ 听写四步法指导 | ⚠️ **听力片段需学员自找素材** |
| 语法（融入精读）| ✓ 引导拆分 + 反馈 | — |
| 精读（翻译三步走）| ✓ 引导 + 反馈 | — |
| **发音** | 主动筛严重问题 | ⚠️ **AI 无法纠音 → 建议 1V1 老师** |
| 4-3-2 | ✓ 嵌入现有训练加时间压迫 | — |
