# Meaning-focused Output

> Four Strands 的第二条线:**通过说和写学习,注意力在传达信息**。产出活动驱动语言真正被调用。

## WHAT:意义焦点的产出性学习

Meaning-focused Output(意义焦点输出)指学习者**为了把信息传达给听众/读者**而去说或写——不是为了练语法、不是为了背单词,就是**有事情要说,有观点要表达**。

典型活动:对话、演讲、讨论、写日记、讲故事、Mini Presentation、Retelling。

和 [[Meaning-focused-Input]] 对称:看 purpose,不看形式。同一次写作,如果目标是"表达一个想法" = Meaning-focused Output;如果目标是"练习使用虚拟语气" = [[Language-focused-Learning]]。

## WHY:输出的三大独特功能(Swain 1995)

Meaning-focused Input 无法替代输出。Swain 给出三条理由:

1. **Noticing / Triggering**——说的时候发现自己"不会说",这个"说不出来"的瞬间直接驱动学习。没有产出压力,就永远不知道自己的缺口在哪。

2. **Hypothesis Testing**——说出来试,看听众反应,看对方听懂没,这是唯一能**验证"我以为的英文"是不是"真的英文"**的方式。

3. **Metalinguistic(Reflective)**——用语言讨论语言本身。翻译练习、给别人讲语法、讨论一个词的用法,都在训练这条。

## WHY:对中国职场学习者为什么核心

Barry 学员的主要瓶颈是**"认识但用不出来"**——被动词汇储备巨大,但输出时调不出来。

[[MCES模型]] 的过渡(从 M→C→E 逐渐走到 M→S)必须**通过大量输出训练**来实现。光读光听不够,脑子里永远停留在"理解层",迈不进"调用层"。

## HOW:五个前提条件(Nation)

1. 写和说的内容**大部分已熟悉**(不要求用全新语言)
2. 主要目标是**传达信息**(不是练形式)
3. 只有少量语言是不熟悉的(跟 input 的 95-98% 已知对称)
4. **可以用策略弥补表达缺口**(绕开、近义替换、解释性表达)
5. **大量产出机会**(没量就没效果)

## HOW:Barry 体系中对应的训练

| Barry 方法 | 对应说明 |
|---|---|
| [[复述-Retelling]] | 听完复述大意,输出端 |
| [[Mini-Presentation训练方法论]] | 框架 + 细节的完整产出 |
| 即兴口语 / 商务翻译 | 限时输出,模拟真实职场场景 |

> ⚠️ **视译不列入 MFO 训练清单**:[[视译训练-Sight-Interpreting]](中→英视译)以传达意思为目的、严格说也算练产出,但**横跨 LFL/MFO,为简化统一归 LFL**(见 [[Four-Strands-训练方法工具箱]] 3.2)。

这些训练覆盖了 Swain 的三大功能:Mini Presentation 覆盖所有三条;翻译天然是 metalinguistic;视译(虽归 LFL)也能覆盖 noticing + hypothesis testing。

## 示例

**是 Meaning-focused Output**:Mini Presentation 讲一个自己亲历的项目故事,目标是让同伴听懂来龙去脉。
**不是 Meaning-focused Output**:写 20 个句子练习被动语态——这是 [[Language-focused-Learning]]。
