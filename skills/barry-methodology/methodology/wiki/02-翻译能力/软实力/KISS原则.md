# KISS 原则

> **Keep It Short and Simple.** 口语沟通的默认原则,职场英语 80% 的场景简单短句 > 复杂长句。它是 [[中到英翻译-三步走]] 输出的英文风格:**切分让它「短」、脱壳让它「简单」**。

## KISS 在三步走里的位置 ⭐

```
[[中到英翻译-三步走]]:
   Step 1 切分 ──→ 让英文「短」(一意一句) ─┐
   Step 2 脱壳 ──→ 让英文「简单」(剥到大白话) ─┤── 合起来 = 输出的英文满足 KISS
   Step 3 出英文小短句 + 逻辑显化 ───────────┘
```

**KISS 不是某一步,而是三步走追求的英文结果。** 切分让"短"成立,脱壳让"简单"成立——所以最后落地的英文小短句天然是 Keep It **Short** and **Simple**。

## 核心观点 / WHAT

**口语沟通中非常常用,默认推荐使用。** 除非场合要求必须使用严肃、复杂的表达,否则都用 KISS。

```
KISS 的本质:一句话,一个意思,一个节拍。
```

**三个"应该":**
- 短句优先 → 每次开口别超过 15 秒
- 简单词优先 → 高频词 > 生僻词
- 清晰结构优先 → 主-谓-宾,别堆从句

## WHY:为什么职场默认 KISS

### 三个现实约束

1. **听众的认知负荷**:会议里同事边听边想,句子越长丢得越多
2. **你自己的认知负荷**:非母语者实时组织长句,10 秒后就掉句
3. **反应速度**:职场沟通是**互动**,不是**朗诵**。KISS 让你快速开口、快速接话

### 为什么"复杂句 = 英文好"是错觉

很多学员以为**用从句、用高级词 = 英文好**,实际上:

- **母语者在工作场合大量用简单句**——他们**能用复杂句,但不选**
- 复杂句是**能力上限的展示**,不是**职场沟通的默认**
- 职场场景里,**听得懂 > 听得惊艳**

## HOW:KISS 执行强度按 CEFR 级别调节

**KISS 不是一刀切**,执行强度随水平调节。

| 档位 | 对应 CEFR | KISS 执行方式 |
|---|---|---|
| **A 档** | A1-A2 | **严格 KISS**:短句为主,一句话一个意思 |
| **B 档** | B1-B2 | **适度 KISS**:可以用连词串联,适当使用从句 |
| **C 档** | C1 及以上 | **灵活处理**:可以用更复杂的句式嵌套信息,但不刻意复杂化 |

### A 档严格 KISS 示例

**场景**:汇报项目进度

> *The project is on track. We finished the design last week. Development starts tomorrow.*

短句为主,每句一个事实。听起来不酷,但**听众 100% 听懂**。

### B 档适度 KISS 示例

同一个 Message,可以升级为:

> *The project is on track—we wrapped up the design last week and we'll kick off the development tomorrow.*

加了破折号连接、加了 and 并列,但**仍然在 KISS 的精神里**:一个思路,清晰结构。

### C 档灵活示例

> *We're on track with the project. Having completed the design phase last week, we're now set to move into development starting tomorrow.*

用了分词结构,但**只在必要时用**——如果场合是跟同事同步进度,前两版反而更合适。

## KISS ≠ 幼稚

**警惕"KISS = 小学生英语"的误解。**

- KISS 是**选择**,不是**能力上限**
- KISS 是**让对方 easier 地接收你的信息**,不是**你 easier 地组织信息**
- 母语者跟同事说话时常用 KISS,写法律文书时才切换到复杂句——**场合驱动,不是习惯**

## KISS 和其他方法论的配合

### KISS ≠ 不用连词

- 连词(but / so / because)是**清晰的逻辑标记**,不违反 KISS
- 违反 KISS 的是**从句嵌套从句**(*which is the thing that, in the context of what we discussed yesterday, might be...*)

详见 [[逻辑连词显化]]。

## 示例:同一 Message 的三档 KISS 水平

**Message**:因为客户反馈不好,所以下一版要调整

| 档位 | 英文 |
|---|---|
| A 档严格 | *Clients were unhappy. So we need to revise the next version.* |
| B 档适度 | *Clients weren't happy with it, so we need to adjust the next version.* |
| C 档灵活 | *Given the pushback we got from clients, we'll need to rework the next version.* |

**三个都合格**——匹配自己水平的就是最好的。
