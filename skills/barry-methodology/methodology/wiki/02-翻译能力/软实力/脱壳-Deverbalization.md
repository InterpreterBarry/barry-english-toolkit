# 脱壳(Deverbalization)

> 脱离文字的束缚,找到文字背后的意思,用大白话讲清楚(像给孩子解释)。**它本身不翻译**——是中到英、英到中都要用的破局动作。

## 核心理念

> **信息/意思是灵魂,语言只是肉身。**
> 翻译不是翻文字,是翻 Message。

- 理想:地道中文 → 地道英文
- 现实:**直接过不去**(字对字必翻译腔)
- 所以:必须**脱壳**——先脱离文字、把背后的意思找出来用大白话讲清。这一步**不涉及翻译**;之后再用英文重新表达(那是三步走的下一步)

## WHAT:脱壳 = 一个动作

**关键认知**:脱壳**只有一个动作**——把文字背后的 Message 找出来。

这个动作**对所有人都一样**。不同的是脱壳之后用什么词来重新表达——**这才是因人而异的**。

```
中文原文 → [脱壳] → 赤裸的 Message → [重新表达] → 英文
         (对所有人一样)              (因人而异)
```

## WHY:为什么要脱壳

**因为中英文不是字对字对应的。** 抓着原文的字直接翻,会产生:
- **翻译腔**(机械对应,不自然)
- **逻辑链断裂**(中文隐含的逻辑没显化)
- **文化违和**(中文表达习惯直接嫁接到英文)

脱壳就是**把意思从文字里解放出来**,让意思本身(而不是原来的字)成为重新表达的对象。

## HOW:脱壳的操作

### 示例 1(Barry 经典案例)

**原文**:供应商和生产线的数据相关性比较薄弱

**① 脱壳**(找意思,所有人一样):**两边的数据对不上**

**② 重新表达成英文**(因人而异,这一步才翻译):
- 基础水平:*The data from A and B don't match.*
- 较高水平:*The supplier data and the production-line data don't really line up.*

> 注:脱壳止于意思本身——如果把"相关性弱"引申成"没用同一套指标",就加入了原文没有的归因;需要补充信息时先确认语境。

## 脱壳在翻译链路中的位置

脱壳是 [[中到英翻译-三步走]] 的第二步:

```
[[切分]](一句中文有几个意思,拆开)
    ↓
脱壳(Deverbalization,剥离文字、找到意思、大白话讲清)    ← 本页
    ↓
[[逻辑连词显化|出英文 + 逻辑显化]](用简单英文小短句说出来,补上逻辑连词)
```

## 脱壳做不到位的典型症状

- 英到中译文**翻译腔重**(例:"如果我看这里,我看到你完成了英语的这个本科学位")
- 中到英译文**中式英语**(例:字对字直译成语/俗语)
- 听起来像**机器翻译**而不是"人话"

修复:回到脱壳阶段,问自己"**他到底想说什么**",忽略他具体用了什么字。
