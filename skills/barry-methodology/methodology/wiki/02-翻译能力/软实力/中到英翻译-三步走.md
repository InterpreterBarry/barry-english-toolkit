# 中到英翻译:三步走

> **切分 → 脱壳 → 逻辑显化**。Barry 教学的核心操作步骤。学员最大痛点 = 字对字翻译,这套三步走是直接纠正这个问题的抓手。

## 核心观点 / WHAT

**任何复杂中文,先按三步降解,再翻英文。**

| 步骤 | 操作 | 说明 |
|---|---|---|
| **Step 1 切分** | 一句中文里有几个意思,拆开 | **一个 message 一句话**,先不翻、先不简化 |
| **Step 2 脱壳** | 剥掉文字外壳,找到每个意思到底想说啥,用大白话讲清(像给孩子解释) | **这一步不翻译**,产出的是赤裸的意思,不是英文 |
| **Step 3 出英文 + 逻辑显化** | 把每个大白话意思用简单英文小短句说出来,再补上中文省略的逻辑词 | 小短句 = 主谓宾;连词 = And / While / Therefore / But… |

整套三步走**最终输出的是英文小短句**。切分让它**短**、脱壳让它**简单**——所以输出天然满足 [[KISS原则|KISS(Keep It Short and Simple)]]。

## WHY:为什么是这三步

### 学员最大痛点 = 字对字翻译

中国学员最容易陷入的死胡同:
- **Word-for-word translation**(字对字)
- **Expression-for-expression**(表达对表达)
- **Sentence structure-for-sentence structure**(句型对句型)

只关注**文字本身**,不关注**文字背后的意思**——这就是软实力训练要纠正的核心问题。

### 中文 vs 英文的两个根本差异

1. **中文一句话塞多个意思,英文一句一个意思**
   - 中文:"我是 2020 年 2 月份转去销售岗的,那个时候正好是薪酬制度从提成制转成目标完成制的关键节点。"
   - 这句中文塞了 4 个信息(转岗时间 / 薪酬制度变化 / 制度具体内容 / 时间点的重要性)
   - 直接对应一句英文:**B1-B2 水平的人组织不出来**,硬凑就是中式英语翻译腔

2. **中文逻辑关系隐含,英文必须显化**
   - 中文里"因为""所以""但是"很多时候不说出来,听者自己悟
   - 英文必须用 because / therefore / however 等连词**明确表达**

三步走分别对应解决:
- Step 1 切分 → 解决"中文一句多意思"
- Step 2 脱壳 → 解决"被原文文字绑架"(抓意思,不抓字)
- Step 3 出英文 + 逻辑显化 → 用简单英文落地 + 解决"逻辑隐含"问题

## HOW:完整操作示例

### 原文

> 我是 2020 年 2 月份转去销售岗的,那个时候正好是薪酬制度从提成制转成目标完成制的关键节点。

### Step 1 切分:找出几个 message

一个 message 一句话——这句中文实际上包含 4 个独立 message:

1. 我是 2020 年 2 月份转去销售岗的
2. 在那个时间点,我们的薪酬制度发生了变化
3. 2020 年之前我们是提成制,之后变成了目标完成制
4. 那个时间点是一个关键节点

### Step 2 脱壳:用大白话把每个意思讲清楚

这一步**不碰英文**——就像给孩子解释,把每个意思用最简单的中文说明白:

1. 我 2020 年 2 月转去做销售了
2. 那时候公司发钱的方式变了
3. 以前是按提成发,后来改成看有没有完成目标
4. 这个时间点挺关键的

### Step 3 出英文 + 逻辑显化

把每个大白话意思用**简单英文小短句**说出来,再补上中文省略的逻辑词:

```
1. I moved to sales in February 2020.
2. Around that time, the way we got paid changed.
3. Before, it was commission-based. After that, it was based on hitting targets.
4. And that timing really mattered.
```

每句都是 B1 水平**开口就能说**的结构,组合起来意思完整。"那个时候正好是关键节点"这种"巧合"关系,需要时用 *Around that time* / *And it just so happened that* 显化出来。

## 脱壳的核心技巧:想给家里孩子解释

> 用什么样的语言?——**先把复杂中文变成简单中文(脱壳),再变成简单英文(Step 3)。**

这是脱壳的具体心法:**多一个中间台阶,不要从复杂中文直接跳英文**。
- ❌ 错误路径:复杂中文 → 直接拼对应英文(陷入字对字)
- ✅ 正确路径:复杂中文 →(脱壳)**简单中文** →(出英文)简单英文

## MCES 模型上的位置

中到英三步走跑的是 **C → E** 这一段:
- Step 1 切分 + Step 2 脱壳 = **重新组织 C**(把复杂中文降成简单大白话,还在中文层,不碰英文)
- Step 3 出英文 + 逻辑显化 = **C → E**(从简单中文落地成简单英文)

详见 [[MCES模型]]。三步走是软实力在 C→E 段做的"换路径"动作——硬实力调不出某个表达时,通过简化和切分让 message 用更简单的英文表达出来。

## 常见误区

| 误区 | 问题 |
|---|---|
| "我水平高了就不用三步走" | 你跟 native 的 gap 永远在,三步走是**长期的底层操作**;水平高只是跑得快(脱壳在脑子里瞬间完成),不是跳过 |
| "英文应该一句话搞定,拆多句不够地道" | 初学者拆多句**远比硬凑长句地道**。水平上来了再合句 |
| "连词太多英文显得啰嗦" | 连词本身不啰嗦,**套层从句才啰嗦**。显化关键逻辑是清晰的 |
