# Communication Outline(演讲准备 4 步)

> 演讲开始之前,必须先回答的**三个听众问题**:Focus / Relevance / KISS。**Communication Outline 4 步走**(Step 0 识别听众 → Step 1 Outcome → Step 2 Relevance → Step 3 Point)。这是半即兴演讲 Content 章节的核心工具。

## 板块定位

[[0-公众演讲板块总览|公众演讲板块]] / [[0-半即兴演讲总览|半即兴演讲]] / **内容(Content)章节的核心起点**。

## WHAT:三个听众抱怨 → 三大原则

听众听演讲时,**心里永远有三个抱怨**:

| 听众抱怨 | 对应原则 | 核心问题 |
|---------|---------|---------|
| 他到底在说什么? | **Focus** | What's the point? |
| 跟我有啥关系? | **Relevance** | Why should I care? |
| 好啰嗦 & 听不懂! | **KISS** | Why so complicated? |

这三个问题如果现场不能回答清楚,演讲基本失败。

## WHY:为什么从这三个开始

搭结构之前必须先回答这三个,否则结构再漂亮也是**自嗨**:

- **Focus** 决定"内容的重心放在哪"
- **Relevance** 决定"听众为什么要听"
- **KISS** 决定"如何不让他们累"

忽略任何一个 → 掉进 [[Performance-Grid|Frustrated]] 象限。

## HOW:Communication Outline 4 步 ⭐

```
Step 0 → Who's the listener?      ← 飞书课件最新版新增
   先识别听众是谁,后面所有思考才有靶子

Step 1 → Outcome
   By the end of the conversation, they will decide/agree to...
   In order to achieve this, they need to:
     KNOW: 1... 2... 3...
     Feel: 1... 2...
     Do:   1... 2...

Step 2 → Relevance
   Why should they care? 1... 2...

Step 3 → Point
   What's your message in one sentence?
```

### Step 0:Who's the listener? ⭐

**这是 Feishu 课件最新版的更新**——把"识别听众"作为 Step 0,**先于 Outcome**。

**为什么 Step 0 重要**:
- Outcome 取决于听众是谁——给老板讲跟给同事讲完全不同
- Relevance 取决于听众关心什么——领导关心结果,执行层关心方法
- Point 取决于听众能接受多少——一句话给老板,可以详细讲给团队

**操作问题清单**:
- 听众是**谁**?(职位 / 角色 / 关注点)
- 他们对话题已经**知道多少**?
- 他们**最关心**什么?
- 他们**最反感**什么?
- 他们想从这次演讲里**带走什么**?

### Focus:Know / Feel / Do

Focus 不只是 Know,还有 **Feel 和 Do**。原因:

> **We make decisions based on our feelings.**

如果只让听众"知道"但没"感觉",他们**不会行动**。所以好演讲永远同时设计三个目标:

- 听众要 **Know** 什么(事实 / 数据 / 框架)
- 听众要 **Feel** 什么(情绪 / 认同 / 紧迫感)
- 听众要 **Do** 什么(下一步行动)

### Relevance:Why should they care?

**最常见的错误**:Speaker 花了 80% 时间讲"我们团队做了什么",完全没讲"**这跟你有什么关系**"。

Relevance 的问法:
- 这个信息对听众的**工作、钱、地位、时间**有什么影响?
- 不听会**错过什么 / 损失什么**?
- 听了能**得到什么 / 避免什么**?

### KISS:Why so complicated?

详见 [[KISS原则]]。核心:**Keep It Simple and Short**。

- 能一句话说清,不要三句
- 能用简单词,不用生僻词
- 能用具体 example,不用抽象概念

## 与后续模块的衔接

Communication Outline 不是终点,而是起点:

```
Communication Outline(Step 0-3)
      ↓
演讲标准结构(Ramp → Roadmap → 3 PoDs → Q&A → Dessert)
      ↓
Techniques(Stories / Metaphors / Active language / Refrain)
      ↓
Delivery(声音 + 外在)
```

**整个板块的底色**是 [[板块核心心态]]——演讲不是表演,是分享。

## 示例:同一个选题的 Outline

**选题**:"推动团队使用新的项目管理工具"

- **Outcome**:By the end, 团队 agree to 切换到新工具
  - Know:旧工具的 3 个痛点、新工具的 3 个优势
  - Feel:旧工具浪费了大量时间,这事**该解决了**
  - Do:下周一开始试用,两周后复盘
- **Relevance**:每人每周能省 4 小时、减少 3 次无效会议
- **Point**:"换工具不是增加工作,是**减少工作**。"
