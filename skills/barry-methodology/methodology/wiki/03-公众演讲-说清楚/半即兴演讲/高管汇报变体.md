# 高管汇报变体

> 面对高管时,[[演讲标准结构|标准演讲结构]]需要**调整顺序**。核心:**先结论,后展开**。

## WHAT:变体结构

**标准结构**:
```
Ramp → Roadmap → 3 PoDs → Q&A → Dessert
```

**高管变体**:
```
Ramp → Roadmap → Executive Summary of 3 PoDs → Q&A explores PoDs → Dessert
```

关键变化:
- **Executive Summary 前置** —— 3 PoDs 在开头被压缩成 1-2 分钟摘要,而不是在 Body 里展开
- **Q&A 顺序灵活** —— 高管想先问哪个 PoD 就展开哪个,不按准备顺序
- **Body 变成"按需展开"** —— 不是线性推进,是**按问题动态跳跃**

## WHY:为什么高管场景要变体

**高管的时间和注意力模式不一样**:

1. **时间稀缺**:他们只愿意给你 5-15 分钟,你铺垫太长他直接打断
2. **信息密度要高**:他们听过无数汇报,先抛出**核心信息**才能抓住他们
3. **他们想主导**:高管习惯通过提问来控制信息获取路径 → Q&A 灵活是尊重他们的工作方式
4. **他们看的是决策点,不是过程**:"So what?" 先答,再补"Why?"

**本质**:高管场景,**你不是演讲者,是答疑者**。

## HOW:两种高管汇报形式

### 关键词提示版(适合自己有把握的场景)

- 只准备**每个 PoD 的核心词 + 数据**
- 完整句子靠现场组织
- **优势**:灵活,能跟着问题跳
- **劣势**:对现场表达能力要求高

### 完整句子版(适合第一次讲或高风险场景)

- 每个 PoD 的 Executive Summary **写完整句子**
- 但仍然不逐字背,要保留自然度
- **优势**:保底,不会跑题
- **劣势**:不如关键词版灵活

## Executive Summary 的写法

这是变体的核心,每个 PoD 用**一句话压缩**:

```
PoD 1:[现状 + 问题]一句话
PoD 2:[解决方案]一句话
PoD 3:[资源需求 / 决策请求]一句话
```

### 示例:Q3 业务汇报

**标准版(Body 展开)**:会讲 3-5 分钟每个 PoD,有数据、有 Story、有 example。

**高管版 Executive Summary**:

> "三件事,90 秒讲完。**第一**,Q3 收入 miss 15%,主要来自 B 端流失。**第二**,我们做了两个动作 —— 重签 Top 20 客户合同,上线 AI 客户成功团队。**第三**,我需要 Q4 批 200 万做市场渗透。Michael,你想先聊哪块?"

## 和金字塔的直接对应

这是 [[Phase2-金字塔结构]] 在高管场景的**精华态**:

- **金字塔顶端(Key Message)** → 开场一句话(结论先行 / BLUF)——占 Ramp 槽位,但此处装的是结论句,不是 [[演讲标准结构|标准 Ramp]] 的开场钩子
- **三个支撑** → Executive Summary 三句话
- **每个支撑的 evidence** → Q&A 时按需展开

**高管汇报 = 金字塔结构的"只讲顶部"**。

## Q&A 主导时的三个准备

既然 Q&A 会主导,准备方法要变:

1. **预判 3 个最可能被追问的 PoD**,提前把数据 / evidence / example 备好
2. **准备"如果他问 X 我就跳到 Y"的路径图**
3. **准备三个可以收尾的 Dessert 版本**,因为你不知道具体几点结束
