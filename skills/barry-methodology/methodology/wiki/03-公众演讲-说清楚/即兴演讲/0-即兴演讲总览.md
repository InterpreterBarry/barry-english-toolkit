# 即兴演讲(Impromptu)· 总览

> 公众演讲板块的**第二大类型**(跟半即兴并列)。**临场被叫上台**的应对——没准备 / 没 PPT / 没稿子兜底,但要讲清楚。**两层结构**:Mindset(心态)+ Structure(5 个口诀框架)。

## 板块定位

```
[[0-公众演讲板块总览|公众演讲板块]]
├── 0. 基本定义
├── ⭐ 板块核心心态(贯穿)
├── 1. 半即兴演讲(Extemporaneous)
└── 2. 即兴演讲(Impromptu) ← 本章节
    ├── 2.1 [[心态-Mindset]] (Dare to be dull / Conversation not performance)
    └── 2.2 [[结构-5框架]] (调用 [[Mental-Models-总览|Mental Models]])
```

## WHAT:即兴 vs 半即兴

| 类型 | 准备程度 | 本质 |
|---|---|---|
| **Extemporaneous(半即兴)** | 大纲 + 关键词 | 主流职场演讲形态(汇报 / 演讲 / 路演) |
| **Impromptu(纯即兴)** | **无准备** | 被点名、临时被问、突发发言 |

**典型 Impromptu 场景**:
- 老板突然说"Barry,你来说两句"
- 会议被点名评论某方案
- 婚礼/聚会上被叫上台 toast(致辞类可直接套 [[WHAT-敬酒词框架]])
- [[Q&A应对|Q&A 被问到意料之外的问题]]
- 跨部门协作会议被要求"分享一下你们组的进展"

## 即兴演讲的核心心法

```
临场被叫上台
   ↓
[[心态-Mindset|2.1 心态调整]]
   ├── Dare to be dull(敢于平淡)
   └── Conversation, not performance(对话,不是表演)
   ↓
[[结构-5框架|2.2 套上一个微结构]](从 5 个口诀框架里选)
   ├── What — So What — Now What
   ├── PREP
   ├── Problem — Solution — Benefit
   ├── Comparison — Contrast — Conclusion
   └── STAR
   ↓
开口
```

## WHY:即兴演讲为什么是通用职场生存能力

raw v1.0 一开始说"不讲",但**实际职场里 Impromptu 的发生频率远高于正式演讲**:
- 每次会议被点名评论 → Impromptu
- Q&A 临场答 → Impromptu
- 突发跨部门同步 → Impromptu

**会半即兴 + 不会即兴 = 在大部分日常场景都靠不住**。

所以 Impromptu **必须讲**——而且 Mindset + Structure **两条腿**。

## 即兴演讲极度依赖 [[板块核心心态|核心心态]]

半即兴有结构、有准备,即使心态紧张也能用结构兜底。**即兴没准备,只剩心态兜底**:

```
心态崩 → 即兴必崩
心态稳 → 即兴有救(套个结构就能讲)
```

所以即兴演讲的**第一步**是 [[心态-Mindset]],不是结构。

## Mindset 简介(详见独立页)

两条心法,**实际就是 [[板块核心心态]] 在即兴场景的具体落地**:

| 心法 | 在板块核心心态中对应 |
|---|---|
| **Dare to be dull** | Congruence(内外一致)—— 不演,真实就行 |
| **Conversation, not performance** | 更大的目标(I'm here to share)—— 对话姿态 |

详见 [[心态-Mindset]]。

## Structure 简介(详见独立页)

**5 个口诀框架**——临场套上一个,就有骨架:

| 框架 | 适用 | Mental Model 链接 |
|---|---|---|
| **What — So What — Now What** | 反思类 / 经验类 | [[What-So-What-Now-What]] |
| **PREP** | 表达观点 / 立场 | [[PREP-Point-Reason-Example-Point]] |
| **Problem — Solution — Benefit** | 提建议 / 解决问题 | [[Problem-Solution-Benefit]] |
| **Comparison — Contrast — Conclusion** | 比较类问题 | [[Comparison-Contrast-Conclusion]] |
| **STAR** | 描述案例 / 经验回顾 | [[STAR-Situation-Task-Action-Result]] |

**这 5 个框架都在 [[Mental-Models-总览|Mental Models 板块]] 的"演讲沟通类框架/即兴演讲框架"分类下**——本板块**调用,不重复内容**。

详见 [[结构-5框架]]。

## 临场操作流程(30 秒)

被点名 → 开始讲之前内心走这三步:

```
1. 听清楚问题 / 主题(必要时复述确认)
        ↓
2. 选一个微结构(从 5 个里挑)
        ↓
3. 想第一句(只想第一句,剩下边讲边想)
        ↓
开口
```

**不要**:
- 开头说"呃我想想"——直接进入第一句
- 说"这个问题问得好"——废话,掉价
- 逐字背答案——你不可能背得出来

## 训练方法:Mini-Presentation

训练 Impromptu 最好的方法是 [[Mini-Presentation训练方法论]]:
- 抽一个随机主题
- 30 秒准备
- 3 分钟讲完
- 重复 + 反馈

**训练量到位之后**,"即兴"就变成了**肌肉记忆**——结构是肌肉记忆,开头句是肌肉记忆,收尾也是肌肉记忆。

## 跟 Q&A 的关系

Q&A 本质是 Impromptu 的子集——区别:
- Q&A 可以**提前准备大部分**(问题可预判,见 [[Q&A应对]] 的 10+10 清单)
- 完全的 Impromptu **没准备**

[[Q&A应对|Q&A 应对法]](Aggressive / Hostile)在 Impromptu 中也通用。

## 跟半即兴演讲的关系

半即兴和即兴**共享心态层**(都依赖 [[板块核心心态]])。
区别在结构:
- 半即兴用 **Ramp → Roadmap → 3 PoDs → Q&A → Dessert**
- 即兴用 **5 个微框架之一**

**学好半即兴会反过来帮你即兴**——半即兴里的结构感、节奏感、Communication Outline 思维,**临场都能用上**。
