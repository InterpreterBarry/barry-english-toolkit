## WHAT:5 个微结构

| 框架 | 结构 | 适用场景 |
|---|---|---|
| **What — So What — Now What** | 咋了 → 那咋了 → 然后呢 | **反思类 / 经验类**(讲一件你经历过的事的意义) |
| **PREP** | Point — Reason — Example — Point | **观点 / 立场**(表达单一观点) |
| **Problem — Solution — Benefit** | 抛痛点 → 给方案 → 讲好处 | **建议 / 解决问题**(给老板提建议) |
| **Comparison — Contrast — Conclusion** | A 特点 → B 特点 → 综合结论 | **比较类**(A vs B 你选哪个) |
| **STAR** | Situation — Task — Action — Result | **案例 / 经验回顾**(讲一个你最有挑战的项目) |

> **5 个框架都在 [[Mental-Models-总览|Mental Models 板块]] 的"演讲沟通类框架/即兴演讲框架"分类下**——本页**调用,不重复展开内容**。每个框架的详情见对应 wiki 页。

## HOW:怎么选框架(快速判断)

被点名后听清问题 → **5 秒内判断属于哪类问题**:

```
听到的问题
   ↓
是观点 / 立场类?           ──→ PREP
   ↓ 不是
是反思 / 经验意义类?       ──→ What-So What-Now What
   ↓ 不是
是建议 / 解决问题类?       ──→ Problem-Solution-Benefit
   ↓ 不是
是比较类(A vs B)?         ──→ Comparison-Contrast-Conclusion
   ↓ 不是
是案例 / 项目经验类?       ──→ STAR
```

**判断关键**:**问题的核心动词**——"你怎么看 / 你的观点(PREP)" / "举个例子(STAR)" / "怎么解决(PSB)" / "选哪个(CCC)" / "你学到什么(WSW-NW)"。

## 例子库 ⭐(来自飞书课件)

5 个真实即兴问题,可对照练习:

### 1. What does it take to become a good leader?

**适用框架**:**PREP**(观点 / 立场类)
- **Point**:好的领导最重要的是**主动倾听**
- **Reason**:因为团队最大的痛点是"老板听不进去"
- **Example**:我之前的老板每次开会都说"任何意见都欢迎",但我们提了三次都没改——后来换的老板每次都先问"我刚才哪里没看到",一年后团队战斗力翻倍
- **Point**:所以**主动倾听比威严更重要**

### 2. Do we need to learn English when AI can help us translate?

**适用框架**:**PREP** 或 **Comparison-Contrast-Conclusion**
- **Comparison(AI 翻译)**:能翻字面 / 24 小时可用 / 免费
- **Contrast(人工英语)**:能感知关系 / 处理微妙语境 / 即时回应
- **Conclusion**:**AI 翻字面,你翻关系**——所以更要学

### 3. Inefficient meetings are draining team energy: What should you do as a manager?

**适用框架**:**Problem — Solution — Benefit**
- **Problem**:80% 的会议没有明确产出
- **Solution**:每次会议必须有议程 + 决策项 + 主持人,**不满足三项不开**
- **Benefit**:每周省 5-10 小时 / 团队士气提升 / 决策更快

### 4. Internal Promotion vs. External Recruitment: How to balance executive selection?

**适用框架**:**Comparison — Contrast — Conclusion**(详见 [[Comparison-Contrast-Conclusion]] 例 1)

### 5. Describe a challenging case you encountered.

**适用框架**:**STAR**(详见 [[STAR-Situation-Task-Action-Result]] 例 1)

## 不是 5 选 1,可以**组合**

复杂问题可以**两个框架嵌套**:
- 主框架 PREP,**Example 部分用 STAR 展开**(描述一个完整案例)
- 主框架 PSB,**Solution 部分用 CCC**(比较两个解决方案)

但**临场不要嵌套太多**——一层主框架 + 一层细节展开就够,**第二层不超过 30 秒**。

## 训练建议

**冷启动**:
1. 选一个框架(比如 PREP)
2. 找 10 个真实问题(从飞书课件 5 题 + 自己工作场景)
3. **每个问题用 PREP 套一遍**(口头说,不写)
4. 练熟一个再换下一个

**进阶**:
- 抽签练习:从 5 个框架里随机抽,30 秒判断 + 30 秒准备 + 2 分钟讲完
- Mini-Presentation 升级:把这套用到 [[Mini-Presentation训练方法论]]

## 跟 Q&A 的关系

[[Q&A应对]] 章节的 Aggressive / Hostile 应对法**跟 5 框架是不同层次**:
- Aggressive / Hostile 应对法:**怎么处理对方**(应对挑衅 / 敌意)
- 5 框架:**怎么组织内容**(临场结构)

实战中两者经常**叠加使用**——先用 Acknowledge / Avoid one-on-one / Reframe 处理对方,再用 PREP / PSB 等组织答案。
