# STAR(Situation — Task — Action — Result)

> **描述案例 / 经验回顾的标准结构**。情境 → 任务 → 行动 → 结果。被 [[wiki/03-公众演讲-说清楚/即兴演讲/结构-5框架|即兴演讲]] 和**面试场景**调用。

## 板块定位

[[Mental-Models-总览|Mental Models]] / 演讲沟通类框架 / **即兴演讲框架(5 个)之一**。

## WHAT:四步结构

| 字母 | 含义 | 一句话 |
|---|---|---|
| **S**ituation | 情境 | 当时是什么背景 |
| **T**ask | 任务 | 你需要解决什么问题 / 达成什么目标 |
| **A**ction | 行动 | 你做了什么(具体动作) |
| **R**esult | 结果 | 最后达成了什么效果(数据 / 反馈 / 学到什么) |

```
Situation ──→ Task ──→ Action ──→ Result
情境          任务        行动         结果
```

## WHY:为什么 STAR 在描述案例时特别好用

**讲案例最容易踩的坑**:
- 跳到结果("我们项目成功了!")—— 听者**没法判断你做了什么**
- 只讲行动 → 听者**没法判断行动有没有效果**
- 没说清情境 → 听者**没法迁移到自己的场景**

STAR 强迫你**走完四步**,让案例**完整、可迁移、有说服力**。

**特别适合的场景**:
- 面试问"讲一个你最有挑战的项目"
- 复盘汇报里讲一个具体案例
- 即兴演讲被问"举个例子"
- 求职简历里描述项目经验

## HOW:操作示例

### 例:面试题"Describe a challenging case you encountered"

| 步骤 | 内容 |
|---|---|
| **Situation** | 去年我所在的销售团队,Q2 业绩下滑了 30%,管理层要求两周内给出方案 |
| **Task** | 我作为团队负责人,需要找到核心原因并制定可执行的恢复计划——而且还要平息团队焦虑 |
| **Action** | 我做了三件事:(1)用 5 Whys 深挖根因,发现是**会议过多挤压销售时间**,不是人力不足;(2)砍掉 50% 的内部会议,把销售时间还给一线;(3)同步给团队开个会,把根因和方案讲清楚,让他们不要焦虑 |
| **Result** | Q3 业绩恢复 + 上涨 10%。**但更重要的副产品**——团队成员都学会了"先找根因再行动",后续遇到类似问题不再恐慌 |

### 例:即兴问"讲一个你最近改变想法的事"

| 步骤 | 内容 |
|---|---|
| **Situation** | 一个月前,我坚信 AI 时代英文翻译能力没用,因为翻译软件已经够强 |
| **Task** | 我跟一位英国客户深度合作,需要判断 AI 翻译能不能替代人工 |
| **Action** | 我故意只用 AI 翻译跟对方沟通了一周,**特意不调整语气和措辞** |
| **Result** | 那一周三次小冲突,**全是因为 AI 没传达对语气和潜台词**。我意识到——**AI 能翻字面,但翻不出关系**。彻底改变了我对"英语沟通"的认知 |

## 跟其他即兴框架的区别

| 框架 | 起点 | 适合 |
|---|---|---|
| **STAR** | 经验 / 案例 | 描述具体经历 |
| [[PREP-Point-Reason-Example-Point]] | 观点 / 立场 | 表达观点 |
| [[Problem-Solution-Benefit]] | 问题 | 提建议 / 解决 |
| [[Comparison-Contrast-Conclusion]] | 对比 | A vs B 类问题 |
| [[What-So-What-Now-What]] | 事件 | 反思类 / 寻找意义 |

**STAR 的独特性**:**它是叙事型框架**——按时间顺序讲故事,有起承转合,听者跟着你"重走一遍"那个场景。

## 跟 [[What-So-What-Now-What]] 的微妙区别

两个都是"讲一个事",但目的不同:

| | STAR | What-So-What-Now-What |
|---|---|---|
| 重心 | **事情本身怎么发生的**(动作和结果) | **事情对你/听者的意义** |
| 适合 | 面试 / 案例汇报 / 描述经验 | 反思 / 启示 / 推荐行动 |
| 结尾 | Result(具体效果) | Now What(下一步行动) |

**类比**:STAR 像**新闻报道**(发生了什么),WSW-NW 像**专栏文章**(这件事意味着什么)。

## 常见误区

| 误区 | 真相 |
|---|---|
| "Result 一定要有数字" | 不一定——**学到什么、改变了什么认知**也是 Result。但**比"项目成功了"更具体的描述**总是更好 |
| "Action 越多越好" | ❌ **关键 Action 抓 1-3 个**就够,讲太多反而模糊重点 |
| "STAR 就是讲故事" | 是讲故事,**但是有结构的故事**——每一步都要回答自己负责的问题,不要随意发挥 |
| "Action 讲了 80%,Result 一句话带过" | 错——听者最关心的是 Result,**Action 和 Result 应该至少 50/50 平分** |
