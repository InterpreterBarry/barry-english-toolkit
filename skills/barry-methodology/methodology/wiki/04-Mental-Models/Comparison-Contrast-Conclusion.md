# Comparison — Contrast — Conclusion(CCC)

> **比较类问题的标准临场结构**。A 怎么样 → B 怎么样 → 综合给结论。被 [[wiki/03-公众演讲-说清楚/即兴演讲/结构-5框架|即兴演讲]] 调用。

## 板块定位

[[Mental-Models-总览|Mental Models]] / 演讲沟通类框架 / **即兴演讲框架(5 个)之一**。

## WHAT:三步结构

| 步骤 | 含义 |
|---|---|
| **Comparison**(比较) | A 选项 / 方案 / 立场的特点 |
| **Contrast**(对比) | B 选项 / 方案 / 立场的特点 |
| **Conclusion**(结论) | 综合两边,给出你的判断或建议 |

```
Comparison ────→ Contrast ────→ Conclusion
A 的特点          B 的特点         综合结论
```

## WHY:为什么比较类问题需要专门的框架

被问到"A vs B 你选哪个"这种问题,**学员最容易犯的错**是:
- **只讲 A 的好**(忽略 B,显得偏激)
- **只讲 A vs B 表面差别**(没给结论)
- **直接给结论,不讲对比**(显得没思考过)

CCC 强迫你**走完三步**:
1. 把 A 讲清楚(包括优势和局限)
2. 把 B 讲清楚(同样优势 + 局限)
3. **基于对比给结论**——结论是从对比里"长出来"的,不是空降的

## HOW:操作示例

### 例 1:Internal Promotion vs. External Recruitment(内部晋升 vs 外部招聘)

| 步骤 | 内容 |
|---|---|
| **Comparison(内部晋升)** | 内部晋升的优势是**对组织文化熟悉、上手快、对老员工有激励作用**。但局限是**视角单一,容易复制现有问题** |
| **Contrast(外部招聘)** | 外部招聘的优势是**带来新视角、能注入新经验和资源**。但局限是**适应期长,有文化冲突风险** |
| **Conclusion** | 所以**关键看公司当前最缺什么**——如果缺执行力和稳定性,内部优先;如果缺创新和外部资源,外部优先。**两条腿走路**,不是非此即彼 |

### 例 2:汇报 vs 邮件,职场怎么选?

| 步骤 | 内容 |
|---|---|
| **Comparison(汇报)** | 当面汇报的优势是**实时反馈、能感知对方情绪、能动态调整**。局限是**留不下文字记录** |
| **Contrast(邮件)** | 邮件的优势是**有迹可循、对方可随时查阅、强迫你想清楚再说**。局限是**没有实时反馈,容易被忽略** |
| **Conclusion** | 所以**重大决策当面+邮件确认,日常进度优先邮件**——根据**信息密度和决策权重**选择媒介 |

## 跟其他即兴框架的区别

| 框架 | 起点 |
|---|---|
| **CCC** | "A 和 B,你选哪个?"(强对比) |
| [[PREP-Point-Reason-Example-Point]] | 表达单一观点 |
| [[Problem-Solution-Benefit]] | 解决具体问题 |
| [[STAR-Situation-Task-Action-Result]] | 描述案例 |
| [[What-So-What-Now-What]] | 反思事件 |

**CCC 的独特性**:**它强制你把两边都讲清楚再下结论**,避免一边倒。这是其他框架做不到的。

## 常见误区

| 误区 | 真相 |
|---|---|
| "我直接说哪个好就行" | 没经过对比的结论**没有说服力**——听众会觉得你"想当然" |
| "Conclusion 一定是 A 或 B" | 不一定——可以是"**两条腿走路**"或"**看具体情境**"——只要是基于前面对比得出的就行 |
| "Comparison 和 Contrast 一定要对称" | 不必——Comparison 讲两个优点 + 一个局限,Contrast 讲三个优点 + 两个局限,**只要把两边讲清楚就行** |
